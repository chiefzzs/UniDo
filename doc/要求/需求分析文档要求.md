1、必须描述需求的背景
2、必须目标用户： 目标用户是系统用户，需要使用系统来完成任务。
3、必须描述目标用户本需求的疼点问题： 目标用户在使用系统时，遇到了什么问题，需要解决。
4、必须描述新的工作流，这个工作流如何解决用户的问题： 新的工作流需要描述用户在使用系统时，需要执行的操作，以及操作的顺序。 
5、必须描述新工作流和原来工作流的区别： 新的工作流需要描述用户在使用系统时，需要执行的操作，以及操作的顺序。
6、列出这个工作流下的所有的场景：按照生命周期，用户由哪些逻辑上时间不连续的操作，每个连续的操作合并为一个场景，需要描述。
7、所有的场景，必须由一个用户界面来触发。
8、描述一个初始的界面原型 
9、每个场景：
   + 必须从界面原型开始
   + 每个场景的输入和输出需要描述，包括输入的参数和输出的结果; 
   + 必须描述这些数据系统是如何一步一步的完整的从输入到达输出的，每个步骤描述一个变化。 
   + 必须描述这些操作后的持久化的数据内容
   + 每个场景只能使用用户的输入，和之前的场景的持久化内容，不能凭空造出数据
    + 针对场景和步骤分析，形成一个次序图，描述用户在使用系统时，需要执行的操作，以及操作的顺序； 
    + 每个场景一个序列图
    + 必须描述每个场景的异常情况： 每个场景的异常情况需要描述，包括异常的触发条件和处理方式。
10、输出领域模型：
   +  基于上述操作场景和相关的数据，输出一个数据模型，描述系统数据关系；先不忙描述数据的详细结构
   + 基于数据的内敛，划分架构层级 ，每个层有哪些服务，持有哪些数据
    + 原子服务： 高内聚低耦合，一个服务一个数据
    + 控制服务： 只对不同的服务聚合，不持有具体的数据
    + 需要采用分层/分服务方式，重新分析上述的序列图在层/服务之间的调用关系。
    + 需要采用分层/分服务方式，重新分析上述的序列图中数据的流向。

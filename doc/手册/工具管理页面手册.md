# 工具管理页面手册

## 概述

工具管理页面是 Trae AI 助手的核心管理界面之一，用于管理系统中的工具定义、配置平台兼容性、设置工具状态等功能。

## 功能说明

### 1. 工具列表展示

| 字段 | 说明 | 数据来源 |
|------|------|----------|
| 工具名称 | 工具的显示名称 | `tool.tool_name` 或 `tool.name` |
| 工具类型 | 工具分类（文件操作、网页搜索等） | `tool.category` 或 `tool.type` |
| 支持操作系统 | 工具支持的平台 | `tool.supported_os` 数组 |
| 支持终端 | 工具支持的终端类型 | `tool.supported_terminals` 数组 |
| 描述 | 工具功能说明 | `tool.description` |
| 状态 | 工具启用/停用状态 | `tool.status` 或 `tool.is_active` |
| 创建时间 | 工具创建日期 | `tool.created_at` |

### 2. 搜索功能

- **输入框位置**: 页面顶部搜索框
- **触发方式**: 实时输入搜索
- **搜索范围**: 工具名称、描述

### 3. 新建工具

点击「+ 新建工具」按钮打开新建弹窗：

| 表单字段 | 类型 | 必填 | 默认值 |
|----------|------|------|--------|
| 工具名称 | 文本输入 | 是 | 空 |
| 工具类型 | 下拉选择 | 否 | websearch |
| 支持操作系统 | 多选框 | 否 | 全选 |
| 支持终端 | 多选框 | 否 | 全选 |
| 描述 | 文本域 | 否 | 空 |
| 配置参数 | JSON文本域 | 否 | {} |
| 状态 | 下拉选择 | 否 | active |

### 4. 编辑工具

点击工具列表中的「编辑」按钮：
- 弹窗标题变为「编辑工具」
- 表单预填充选中工具的现有数据
- 保存后更新工具信息

### 5. 删除工具

点击工具列表中的「删除」按钮：
- 弹出确认对话框
- 确认后删除该工具

## 前端组件说明

### 组件位置

`static_src/js/components/admin/ToolPageComponent.js`

### 核心方法

| 方法名 | 功能说明 |
|--------|----------|
| `loadTools()` | 从后端加载工具列表 |
| `handleSearch()` | 处理搜索请求 |
| `getToolTypeName(type)` | 转换工具类型显示名称 |
| `getPlatformLabels(osArray)` | 格式化操作系统标签 |
| `getTerminalLabels(terminalArray)` | 格式化终端标签 |
| `getToolStatus(tool)` | 获取工具状态 |
| `showCreateModal()` | 显示新建弹窗 |
| `handleEdit(tool)` | 显示编辑弹窗 |
| `handleSave()` | 保存工具（新建/更新） |
| `handleDelete(id)` | 删除工具 |

### 事件监听

组件通过 `EventBus` 监听以下事件：

| 事件名 | 触发时机 |
|--------|----------|
| `tool:loaded` | 工具数据加载完成 |
| `tool:created` | 新工具创建成功 |
| `tool:updated` | 工具更新成功 |
| `tool:deleted` | 工具删除成功 |

## 注意事项

1. **数据持久化**: 工具数据通过 `ToolManager` 服务与后端交互
2. **状态管理**: 使用 `StateManager` 进行客户端状态缓存
3. **事件驱动**: 通过 `EventBus` 实现组件间通信
4. **空值处理**: 当 `supported_os` 或 `supported_terminals` 为空数组时显示「全部平台」/「全部终端」
5. **字段兼容**: 支持后端返回的 `tool_name`/`name`、`category`/`type`、`is_active`/`status` 等多种字段命名方式

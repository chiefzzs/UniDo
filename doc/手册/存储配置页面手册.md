# 存储配置页面手册

## 概述

存储配置页面用于管理系统的持久化配置，包括实体存储配置和事件存储配置两个标签页。

## 页面结构

│  │ 📦 实体存储配置     │  │ ⚡ 事件存储配置     │           │
│  └─────────────────────┘  └─────────────────────┘           │
├─────────────────────────────────────────────────────────────┤
│  [实体存储配置标签页内容]                                     │

## 功能说明

### 1. 实体存储配置

控制哪些实体类型需要持久化存储。

| 字段 | 说明 |
|------|------|
| 实体类型 | 实体的类型标识 |
| 描述 | 实体的描述信息 |
| 持久化 | 是否持久化该实体类型 |
| 更新时间 | 最后更新时间 |

### 2. 事件存储配置

控制哪些事件类型需要存储到项目时间线。

| 功能 | 说明 |
|------|------|
| 事件类型筛选 | 按事件类别筛选 |
| 仅显示停用项 | 只显示未启用的事件 |
| 重置为默认 | 恢复默认配置 |
| 启用/禁用全部 | 批量操作 |

### 3. 事件分类

| 分类 | 说明 |
|------|------|
| project | 项目事件 |
| session | 会话事件 |
| dialog | 对话事件 |
| message | 消息事件 |
| llm | LLM事件 |
| tool | 工具事件 |
| task | 任务事件 |
| round | 轮次事件 |
| client | 客户端事件 |
| system | 系统事件 |

## 数据结构

### 实体存储配置

```json
{
  "entity_type": "dialog",
  "description": "对话实体",
  "persist": true,
  "updated_at": "2024-01-01T00:00:00Z"
}
```

### 事件存储配置

```json
{
  "event_type": "project.created",
  "description": "项目创建事件",
  "persist": true
}
```

## 事件监听

| 事件 | 触发时机 |
|------|----------|
| `storage:loaded` | 实体存储配置加载完成 |
| `storage:error` | 存储配置加载失败 |
| `eventStorage:loaded` | 事件存储配置加载完成 |
| `eventStorage:updated` | 事件存储配置更新 |
| `eventStorage:batch_updated` | 批量更新完成 |
| `eventStorage:reset` | 重置为默认 |
| `eventStorage:error` | 事件存储配置错误 |

## 核心方法

### 实体存储配置

| 方法 | 功能 |
|------|------|
| `loadEntityStorages()` | 加载实体存储配置 |
| `toggleEntityPersist(storage)` | 切换实体持久化状态 |
| `saveEntityChanges()` | 保存实体配置更改 |
| `discardEntityChanges()` | 放弃实体配置更改 |

### 事件存储配置

| 方法 | 功能 |
|------|------|
| `loadEventStorages()` | 加载事件存储配置 |
| `toggleEventPersist(event)` | 切换事件存储状态 |
| `saveEventChanges()` | 保存事件配置更改 |
| `discardEventChanges()` | 放弃事件配置更改 |
| `resetEventStorage()` | 重置为默认配置 |
| `toggleAllEvents()` | 启用/禁用全部事件 |

## 使用流程

1. **选择标签页**: 选择实体存储配置或事件存储配置
2. **修改配置**: 通过开关切换状态
3. **批量操作**: 事件存储支持批量操作
4. **保存更改**: 点击保存按钮确认更改

## 注意事项

1. 更改不会立即生效，需要点击保存按钮
2. 可以放弃未保存的更改
3. 事件存储配置可以重置为默认值
4. 建议只存储必要的事件类型以优化性能

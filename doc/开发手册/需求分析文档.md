# 需求分析文档

## 文档概述

本文档围绕用户操作场景，描述每个场景下的系统内部数据流和控制流，以及用户与系统间的交互方式。

---

## 用户操作场景概览

| 场景编号 | 场景名称 | 用户目标 | 涉及页面 |
|----------|----------|----------|----------|
| S01 | 系统启动 | 启动道一服务 | 命令行 |
| S02 | 模型配置 | 配置AI大模型API | 模型管理页面 |
| S03 | 工作区配置 | 配置本地代码目录 | 工作区管理页面 |
| S04 | 项目创建 | 关联模型和工作区 | 项目管理页面 |
| S05 | 会话创建 | 创建对话会话 | 对话页面侧边栏 |
| S06 | 用户对话 | 与AI助手交流 | 对话页面主区域 |
| S07 | 工具执行 | AI调用工具完成任务 | 对话页面 |
| S08 | 提示词管理 | 管理提示词模板 | 提示词管理页面 |
| S09 | 工具管理 | 查看和管理工具配置 | 工具管理页面 |
| S10 | 存储配置 | 配置数据持久化 | 存储配置页面 |

## 场景详细描述

### S01 - 系统启动场景

#### 用户目标
启动道一后端服务，使前端页面可访问。

#### 用户交互流程

```
用户操作                    系统响应
────────────────────────────────────────────────────────────
1. 打开命令行              → 系统等待输入
2. 输入 python run.py      → 系统解析启动命令
                           → 加载配置文件
                           → 初始化各层服务
                           → 启动HTTP服务器(8000端口)
                           → 启动WebSocket服务
                           → 打印启动日志
3. 打开浏览器              → 访问 http://localhost:8000
                           → 前端页面加载完成
```

#### 系统内部数据流

```
┌─────────────────────────────────────────────────────────────────┐
│                        启动数据流                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │

│     ▼                                                           │
│  ┌──────────────────┐                                           │
│  │   APIServer      │  ← L4 Gateway Layer                       │

│  └────────┬─────────┘                                           │
│           │                                                     │
│           ▼                                                     │
│  ┌──────────────────┐     ┌──────────────────┐                  │
│  │ _init_event_bus  │ →   │ PersistenceService│ ← L1b           │
│  │ _persistence     │     │ 初始化            │                  │
│  └────────┬─────────┘     └──────────────────┘                  │
│           │                                                     │
│           ▼                                                     │
│  ┌──────────────────┐     ┌──────────────────┐                  │
│  │ LLMExecution     │ →   │ LLMClient        │ ← L1c            │

└─────────────────────────────────────────────────────────────────┘
```

#### 系统内部控制流

```
启动控制流：
1. run.py → 创建 APIServer 实例
2. APIServer.__init__() →
   a. 创建 FastAPI 应用
   b. _init_event_bus_persistence() → 初始化事件总线持久化
   c. _init_event_console_printer() → 初始化事件日志打印
   d. _init_websocket_plugin() → 初始化WebSocket事件插件
   e. _setup_middleware() → 设置中间件（日志、认证、CORS）
   f. _setup_routes() → 注册路由（项目、会话、消息、配置等）
   g. _setup_static_files() → 挂载静态文件目录
3. uvicorn.run() → 启动HTTP服务器
```

#### 关键数据结构

**启动配置数据：**
```json
{
  "host": "0.0.0.0",
  "port": 8000,
  "title": "AI Agent System API",
  "version": "1.0.0"
}
```

**事件总线配置：**
```python
class EventBusConfig:
    persistence_enabled: True      # 启用持久化
    persist_full_payload: True     # 持久化完整payload
    memory_cache_enabled: True     # 启用内存缓存
```

### S02 - 模型配置场景

#### 用户目标
配置AI大模型API（如OpenAI、Anthropic），使系统能够调用LLM服务。

#### 用户交互流程

```
用户操作                    系统响应
────────────────────────────────────────────────────────────
1. 点击"工具菜单"          → 显示管理菜单列表
2. 点击"模型管理"          → 跳转到模型管理页面
                           → 加载已配置的模型列表
3. 点击"新建模型"          → 显示新建模型表单
4. 填写表单：              → 表单数据暂存
   - 模型名称              →
   - API类型               →
   - API地址               →
   - API密钥               →
   - 模型名称              →
5. 点击"保存"              → 发送POST请求
                           → 后端验证数据
                           → 保存模型配置
                           → 返回成功响应
                           → 前端更新模型列表
```

#### 系统内部数据流

```
┌─────────────────────────────────────────────────────────────────┐
│                      模型配置数据流                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  前端 (Vue.js)                                                  │
│     │                                                           │
│     │ POST /api/models                                          │
│     │ {name, api_type, api_address, api_key, model_name}        │
│     ▼                                                           │
│  ┌──────────────────┐                                           │
│  │ L4a HTTP Gateway │                                           │
│  │ model_routes.py  │                                           │
│  └────────┬─────────┘                                           │
│           │                                                     │
│           ▼                                                     │
│  ┌──────────────────┐                                           │
│  │ L2a Project      │  ← 验证配置数据                            │
│  │ Config Service   │                                           │
│  └────────┬─────────┘                                           │
│           │                                                     │
│           ▼                                                     │
│  ┌──────────────────┐                                           │
│  │ L1b Persistence  │  ← 保存模型配置                            │
│  │ Service          │     storage: 'models'                     │
│  └────────┬─────────┘                                           │
│           │                                                     │
│           ▼                                                     │
│  ┌──────────────────┐                                           │
│  │ EventBus         │  ← 发布 MODEL_CREATED 事件                 │
│  │ publish()        │                                           │
│  └────────┬─────────┘                                           │
│           │                                                     │
│           ▼                                                     │
│  ┌──────────────────┐                                           │
│  │ WebSocket        │  ← 推送事件到前端                          │
│  │ Server           │                                           │
│  └──────────────────┘                                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 系统内部控制流

```
模型配置控制流：
1. ModelPageComponent.vue → 用户填写表单，点击保存
2. ApiClient.post('/api/models') → 发送HTTP请求
3. model_routes.py → create_model() 处理请求
4. ProjectConfigService → create_model_config() 验证并创建
5. PersistenceService → save('models', model_data) 持久化
6. EventBus → publish(MODEL_CREATED, payload) 发布事件
7. WebSocketServer → push_to_connections() 推送到前端
8. ModelPageComponent → 更新模型列表显示
```

#### 关键数据结构

**模型配置请求：**
```json
{
  "name": "OpenAI GPT-4",
  "api_type": "openai",
  "api_address": "https://api.openai.com/v1",
  "api_key": "sk-xxx...",
  "model_name": "gpt-4",
  "max_tokens": 2048,
  "temperature": 0.7
}
```

**模型配置响应：**
```json
{
  "model_id": "model-xxx",
  "name": "OpenAI GPT-4",
  "api_type": "openai",
  "status": "active",
  "created_at": "2026-06-05T10:00:00"
}
```

### S06 - 用户对话场景（核心场景）

#### 用户目标
与AI助手进行对话交流，获取帮助完成编程任务。

#### 用户交互流程

```
用户操作                    系统响应
────────────────────────────────────────────────────────────
1. 选择会话              → 加载会话历史消息
2. 输入问题              → 输入框暂存用户输入
   "帮我优化这段代码"    →
3. 点击发送              → 发送WebSocket消息
                         → 后端接收用户输入
                         → 创建用户消息
                         → 调用LLM服务
                         → 流式返回AI响应
                         → 前端实时显示响应
4. 查看响应              → 消息列表更新
   - 思考过程            → ThinkBlock显示
   - 推理过程            → ReasonBlock显示
   - 文本回复            → TextBlock显示
   - 工具调用            → ToolCard显示
5. 继续对话              → 循环步骤2-4
```

#### 系统内部控制流

```
用户对话控制流：
1. ChatInputComponent → 用户输入，点击发送
2. WSClient.send() → 发送WebSocket消息
3. WebSocketServer._handle_message() → 接收消息
4. DialogueService.process_dialogue() → 处理对话
   a. DialogService.create_dialog() → 创建对话
   b. MessageService.create_message() → 创建用户消息
   c. EventBus.publish(MESSAGE_CREATED) → 发布消息事件
5. BaseExecutionService.execute_with_recursive_llm() → 执行LLM循环
   a. DialogueBasedLLMService.call_llm() → 调用LLM
   b. LLMExecutionService.execute_stream() → 流式执行
   c. LLMClient.call() → 调用外部API
   d. EventBus.publish() → 发布流式事件
      - llm.call_thinking → 思考过程
      - llm.call_reasoning → 推理过程
      - llm.call_text → 文本内容
      - llm.call_completed → 完成事件
6. WebSocketServer.push_event() → 推送事件到前端
7. ChatManager.handleWebSocketMessage() → 处理事件
8. AssistantMessageComponent → 更新显示
```

#### 用户目标

#### 用户交互流程

```

   "读取src/main.py"     →
                         → LLM返回tool_calls
                         → 系统解析工具调用
                         → 执行工具
                         → 返回工具结果
```

| 事件类型 | 发布时机 | 携带数据 |
|----------|----------|----------|
| `system.initialized` | 系统启动完成 | 无 |

| `llm.request_sent` | LLM请求发送 | request_id, dialog_id |

| `llm.call_reasoning` | LLM推理过程 | reasoning片段 |
| `llm.call_text` | LLM文本输出 | content片段 |
| `llm.call_completed` | LLM调用完成 | finish_reason |
| `tool.call_started` | 工具调用开始 | call_id, tool_name |
| `tool.execution_output` | 工具执行输出 | output片段 |
| `tool.call_completed` | 工具调用完成 | result, status |
| `model.created` | 模型配置创建 | model_id, name |
| `workspace.created` | 工作区配置创建 | workspace_id, path |
| `project.created` | 项目创建 | project_id, name |

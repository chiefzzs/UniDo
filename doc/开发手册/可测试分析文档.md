# 可测试分析文档

## 文档概述

本文档描述道一（UniDo）系统各层级的独立测试策略（**向下不允许打桩**），以及如何验证测试目录 `src/data/test/` 中的数据正确性。

---

## 测试策略原则

### 核心原则：向下不打桩

**定义**：上层测试调用下层服务时，必须使用真实实现，**不允许**使用 Mock 或 Stub 替代。

**理由**：
- 确保测试的真实性和可靠性
- 验证各层之间的接口契约
- 发现层间集成问题
- 符合"测试应该像生产环境一样运行"的理念

**例外情况**：
- 外部依赖（如第三方 API）可以打桩
- 不可控的外部系统（如网络、文件系统状态）可以打桩

## 各层测试策略

### L1 基础设施层测试

#### 测试目标
验证基础设施服务的基本功能和正确性。

#### 测试策略
- 直接测试，无下层依赖
- 使用真实的文件系统进行持久化测试
- 使用 Mock 替代外部 API（LLM Client）

#### 测试数据验证

**测试文件**: `src/data/test/*.json`

**验证内容**:

| 数据文件 | 验证项 | 验证方法 |
|----------|--------|----------|
| `events.json` | 事件记录格式正确 | 检查必需字段（event_type, payload, correlation_id） |
| `llm_calls.json` | LLM调用记录完整 | 验证 request/response 结构、usage 统计 |
| `tool_definitions.json` | 工具定义完整 | 检查参数定义、平台兼容性字段 |
| `api_requests.json` | API请求记录正确 | 验证 HTTP 方法、路径、状态码 |

#### 测试用例示例

```python
def test_persistence_service_save_and_load():
    """测试持久化服务的保存和加载功能"""
    # 准备测试数据
    test_data = {
        "record_id": "test-123",
        "name": "Test Record",
        "created_at": datetime.now().isoformat()
    }
    
    # 执行保存
    result = persistence_service.save('test_records', test_data)
    
    # 验证保存成功

    # 执行加载
    loaded_data = persistence_service.list('test_records')
    
    # 验证数据完整性
    assert len(loaded_data) >= 1
    assert loaded_data[-1]['record_id'] == 'test-123'
    assert loaded_data[-1]['name'] == 'Test Record'
```

## 回放模式测试

### 测试目标
验证回放模式下系统能够正确使用历史数据，不产生新的 LLM 调用，确保对话流程的可复现性。

### 回放模式原理

```
│                                                             │

│  └────────┬─────────┘      └────────┬─────────┘             │
│           │                        │                        │
│           ▼                        ▼                        │
│  ┌──────────────────────────────────────────────────────┐   │
│  │           WebSocket 消息路由                         │   │
│  │   ┌────────────────────────────────────────────┐    │   │
│  │   │  判断是否为回放模式                          │    │   │

│  │                        │                            │   │
│  │              ┌─────────┴─────────┐                  │   │

│  │   ┌────────────────┐   ┌────────────────┐           │   │

│  │   └───────┬────────┘   └───────┬────────┘           │   │

```

### 测试策略

**核心规则**：回放模式下发送消息不应产生新的 LLM 调用记录。

**测试方法**：
1. 切换到回放模式
2. 发送消息
3. 验证 `llm_calls.json` 文件大小无明显增加
4. 验证响应来自历史数据

### 测试用例示例

```typescript

import * as path from 'path';

const LLM_CALLS_PATH = path.join(TEST_DATA_DIR, 'llm_calls.json');

      await page.waitForTimeout(1500);
    }

    // 记录发送前文件大小
    let fileSizeBefore = 0;
    if (fs.existsSync(LLM_CALLS_PATH)) {
      const content = fs.readFileSync(LLM_CALLS_PATH, 'utf-8');
      fileSizeBefore = content.length;
    }

    // 发送消息
    const input = page.locator('input[type="text"]');
    await input.fill('回放模式测试消息');
    await page.keyboard.press('Enter');

    // 等待响应
    await page.waitForTimeout(5000);

    // 验证文件大小无明显增加
    if (fs.existsSync(LLM_CALLS_PATH)) {
      const contentAfter = fs.readFileSync(LLM_CALLS_PATH, 'utf-8');
      const fileSizeAfter = contentAfter.length;
      const sizeDiff = fileSizeAfter - fileSizeBefore;
      
      // 回放模式下不应有大量数据写入（允许少量元数据）
      expect(sizeDiff).toBeLessThan(500);
    }
  });

  test('录制模式下发送消息应新增 llm_calls.json 数据', async ({ page }) => {
    await page.goto('/');

    await toggle.waitFor();
    
    const isRecording = await toggle.getAttribute('class');
    if (!isRecording.includes('active')) {
      await toggle.click();
      await page.waitForTimeout(1500);

    // 记录发送前文件大小
    let fileSizeBefore = 0;
    if (fs.existsSync(LLM_CALLS_PATH)) {
      const content = fs.readFileSync(LLM_CALLS_PATH, 'utf-8');
      fileSizeBefore = content.length;
    }

    // 发送消息
    const input = page.locator('input[type="text"]');
    await input.fill('录制模式测试消息');
    await page.keyboard.press('Enter');

    // 等待响应
    await page.waitForTimeout(10000);

    // 验证文件大小增加（LLM调用记录）
    if (fs.existsSync(LLM_CALLS_PATH)) {
      const contentAfter = fs.readFileSync(LLM_CALLS_PATH, 'utf-8');
      const fileSizeAfter = contentAfter.length;
      const sizeDiff = fileSizeAfter - fileSizeBefore;
      
      // 录制模式下应有明显数据写入
      expect(sizeDiff).toBeGreaterThan(1000);
    }

  test('回放模式下历史消息正确加载', async ({ page }) => {
    await page.goto('/');

    // 切换到回放模式
    const toggle = page.locator('.replay-toggle');
    await toggle.waitFor();
    
    const isRecording = await toggle.getAttribute('class');

      await page.waitForTimeout(1500);
```

### 回放模式数据验证

**验证测试数据目录中的数据是否正确用于回放**：

```typescript
import * as fs from 'fs';
import * as path from 'path';

function validateReplayData(testDataDir: string): { valid: boolean; errors: string[] } {
  const errors: string[] = [];
  
  // 检查必需的回放数据文件
  const requiredFiles = [
    'llm_calls.json',
    'messages.json', 
    'dialogs.json',
    'events.json'
  ];

  for (const file of requiredFiles) {
    const filePath = path.join(testDataDir, file);
    if (!fs.existsSync(filePath)) {
      errors.push(`缺少回放数据文件: ${file}`);
      continue;
    }

    try {
      const content = fs.readFileSync(filePath, 'utf-8');
      const data = JSON.parse(content);
      
      if (!Array.isArray(data)) {
        errors.push(`${file} 格式错误: 应为数组`);
      } else if (data.length === 0) {
        errors.push(`${file} 为空，无法进行回放测试`);
      }
    } catch (e) {
      errors.push(`${file} 解析失败: ${e}`);
    }
  }

  return {
    valid: errors.length === 0,
    errors
  };
}

// 使用示例
const result = validateReplayData('src/data/test');
if (!result.valid) {
  console.error('回放数据验证失败:');
  result.errors.forEach(err => console.error(`  - ${err}`));
}
```

### 回放测试数据结构

| 字段 | 类型 | 说明 |

| `dialog_id` | string | 关联对话ID |

| 不同 `dialog_id` + 相同请求内容 | 低 | 跨会话匹配 |

## 测试数据结构验证

### 数据格式规范

#### 1. 消息数据 (`messages.json`)

```json
{
  "message_id": "msg-xxx",
  "dialog_id": "dialog-xxx",
  "role": "user|assistant|tool",
  "content": "...",
  "metadata": {},
  "platform_info": {
    "os_type": "windows|linux|macos",
    "terminal_type": "powershell|bash|zsh"
  },
  "created_at": "2026-06-05T10:00:00"
}
```

**验证规则**:
- `message_id` 必须以 `msg-` 开头
- `role` 只能是 `user`、`assistant`、`tool`
- `created_at` 必须是 ISO 8601 格式
- `platform_info` 对于 `user` 角色是必需的

#### 2. 工具定义数据 (`tool_definitions.json`)

```json
{
  "tool_id": "T01",
  "tool_name": "ToolName",
  "category": "...",
  "description": "...",
  "parameters": {
    "param_name": {
      "type": "string|number|boolean|array|object",
      "description": "...",
      "required": true|false
    }
  },
  "tool_type": "native|async|task_group",
  "supported_os": ["windows", "linux", "macos"],
  "supported_terminals": ["powershell", "bash"],
  "created_at": "2026-06-05T10:00:00"
}
```

**验证规则**:
- `tool_id` 必须以 `T` 开头后跟数字
- `parameters` 中每个参数必须有 `type` 和 `description`
- `supported_os` 和 `supported_terminals` 不能为空数组
- `tool_type` 只能是 `native`、`async`、`task_group`

#### 3. LLM调用数据 (`llm_calls.json`)

```json
{
  "call_id": "call-xxx",
  "dialog_id": "dialog-xxx",
  "model_config_id": "...",
  "request_id": "req-xxx",
  "request": {
    "model_name": "...",
    "messages": [],
    "temperature": 0.7,
    "max_tokens": 2048,
    "stream": true|false,
    "tools": []
  },
  "response": {
    "content": "...",
    "finish_reason": "stop|tool_calls|error",
    "tool_calls": [],
    "usage": {
      "prompt_tokens": 100,
      "completion_tokens": 50,
      "total_tokens": 150
    }
  },
  "created_at": "2026-06-05T10:00:00"
}
```

**验证规则**:
- `finish_reason` 只能是 `stop`、`tool_calls`、`error`
- 如果 `finish_reason` 是 `tool_calls`，必须有 `tool_calls` 数组
- `usage` 字段必须包含 `prompt_tokens`、`completion_tokens`、`total_tokens`

#### 4. 事件数据 (`events.json`)

```json
{
  "record_id": "event-xxx",
  "event_type": "message.created|llm.call_text|tool.call_started",
```

```python
def test_event_publishing():
    """测试事件发布"""
    events_received = []
    
    # 订阅事件
    def handler(event):
        events_received.append(event)
    
    subscription = event_bus.subscribe('message.created', handler)
    
    # 执行操作
    message_service.create_message(
        dialog_id="dialog-test",
        role="user",
        content="Hello"
    )
    
    # 验证事件发布
    assert len(events_received) == 1
    assert events_received[0].event_type == 'message.created'
    
    # 清理订阅
```

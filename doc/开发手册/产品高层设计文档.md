#### 关键数据字段变化追踪

**阶段1：用户输入 → WebSocket消息**

```
用户输入:
  content: "帮我优化src/main.py的性能"
  
WebSocket消息:
  {
    action: "send_message",
    session_id: "sess-xxx",
    content: "帮我优化src/main.py的性能"  ← 无变化
  }
```

**阶段2：WebSocket → Message对象**

```
MessageService.create_message() 输入:
  role: 'user'
  content: "帮我优化src/main.py的性能"
  
Message对象输出:
  {
    message_id: "msg-xxx",
    dialog_id: "dialog-xxx",
    role: "user",
    content: "<system-reminder>\n操作系统类型: windows\n终端类型: powershell</system-reminder>\n<user_input>\n帮我优化src/main.py的性能\n</user_input>",  ← 添加平台信息和标签
    platform_info: {os_type: "windows", terminal_type: "powershell"}  ← 新增字段
  }
```

**阶段3：Message → LLMRequest**

```
LLMRequest对象:
  {
    model_name: "gpt-4",
    messages: [
      {
        role: "user",
        content: [{"type": "text", "text": "..."}]  ← 转换为数组格式
      }
    ],
    tools: [
      {
        type: "function",
        function: {
          name: "read_file",
          description: "读取文件内容",
          parameters: {...}
        }
      }
    ],  ← 新增工具定义
    tool_choice: "auto"
  }
```

**阶段4：LLM响应 → tool_calls解析**

```
LLM响应:
  {
    content: "",
    tool_calls: [
      {
        id: "call-xxx",
        type: "function",
        function: {
          name: "read_file",
          arguments: '{"file_path": "src/main.py"}'  ← JSON字符串
        }
      }
    ],
    finish_reason: "tool_calls"  ← 标识需要工具调用
  }
  
解析后的工具调用:
  {
    call_id: "call-xxx",
    tool_name: "read_file",
    parameters: {file_path: "src/main.py"}  ← 解析后的对象
  }
```

**阶段5：工具执行 → tool消息**

```
ToolExecutor输出:
  {
    call_id: "call-xxx",
    tool_name: "read_file",
    output: "# main.py content...",
    status: "success"
  }
  
Message对象(role='tool'):
  {
    message_id: "msg-tool-xxx",
    role: "tool",
    content: "工具执行结果: # main.py content...",  ← 工具输出
    metadata: {tool_call_id: "call-xxx", tool_name: "read_file"}
  }
```

**阶段6：再次LLM调用 → 最终响应**

```
LLMRequest(messages包含tool结果):
  {
    messages: [
      {role: "user", content: "..."},
      {role: "assistant", tool_calls: [...]},
      {role: "tool", content: "工具执行结果..."}  ← 新增
    ]
  }
  
LLM响应:
  {
    content: "我分析了src/main.py的内容，建议从以下几个方面优化...",
    finish_reason: "stop"  ← 循环结束标识
  }
```

## 关键数据字段变化总结

### 用户输入 → Message → LLMRequest

| 阶段 | 字段 | 变化 |
|------|------|------|
| 用户输入 | content | 原始输入 |
| Message | content | 添加平台信息标签 |
| Message | platform_info | 新增字段 |
| LLMRequest | messages[].content | 转换为数组格式 |
| LLMRequest | tools | 新增工具定义列表 |

### LLM响应 → tool_calls → ToolResult

| 阶段 | 字段 | 变化 |
|------|------|------|
| LLM响应 | tool_calls | JSON字符串格式 |
| 解析后 | parameters | 转换为对象格式 |
| ToolResult | output | 工具执行输出 |
| Message(role='tool') | content | 工具执行结果 |

### 流式响应 → 前端显示

| 阶段 | 字段 | 变化 |
|------|------|------|
| SSE Chunk | delta | 单个片段 |
| StreamChunk | chunk_type | thinking/reasoning/text |
| Event | action | llm.call_xxx |
| WebSocket | data.content | 片段内容 |
| 前端 | 显示 | 累加拼接 |

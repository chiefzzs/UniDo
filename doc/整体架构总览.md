# 整体架构总览

## 1. 系统概述

本系统是一个基于大语言模型的智能对话应用，采用分层架构设计，支持多种对话场景和工具调用。

### 1.1 核心目标
- 提供流畅的对话体验
- 支持任务执行和代码开发
- 实现记忆和状态管理
- 提供可扩展的工具系统

### 1.2 设计原则
- **分层架构**：清晰的层次划分
- **事件驱动**：组件间通过事件通信
- **领域驱动**：基于DDD设计实体
- **可扩展性**：支持新增场景和工具

| 层级 | 名称 | 职责 | 组件 |

| L1 | 基础设施层 | 基础服务 | 配置、持久化、LLM、事件 |

| L5 | UI应用层 | 用户界面 | 对话、侧边栏、配置、任务面板 |

## 3. 核心实体关系

### 3.1 实体关系图

```
Project (聚合根)
  │
  ├─ sessions: List[Session]
  │
  ├─ workspace_id ──→ WorkspaceConfig
  │
  └─ model_config_id ──→ ModelConfig

Session (聚合根)
  │
  ├─ messages: List[Message]
  │
  └─ task_group_id ──→ TaskGroup (可选)

TaskGroup
  │
  ├─ tasks: List[Task]
  │
  ├─ parent_group_id ──→ TaskGroup (可选)
  │
  └─ sub_task_groups: List[TaskGroup]

Task
  │
  └─ tool_name ──→ Tool

Message
  │
  └─ tool_call_id ──→ ToolCall
```

### 3.2 实体职责说明

| 实体 | 职责 | 生命周期 |
|-----|------|---------|
| Project | 项目级配置和会话管理 | 创建 → 活跃 → 归档/删除 |
| Session | 会话级对话和任务管理 | 创建 → 活跃 → 归档/删除 |
| Message | 对话消息存储 | 创建 → 更新 → 删除 |
| TaskGroup | 任务组管理 | 创建 → 执行 → 完成/失败 |
| Task | 单个任务执行 | 创建 → 执行 → 完成/失败 |

## 5. 事件驱动架构

### 5.1 事件类型汇总

| 事件类别 | 事件类型 | 触发时机 |
|---------|---------|---------|
| 项目事件 | project.created | 项目创建成功 |
| | project.updated | 项目更新成功 |
| | project.deleted | 项目删除成功 |
| 会话事件 | session.created | 会话创建成功 |
| | session.updated | 会话更新成功 |
| | session.deleted | 会话删除成功 |
| 消息事件 | message.created | 消息创建成功 |
| | message.updated | 消息更新成功 |
| LLM事件 | llm.request_sent | LLM请求发送 |
| | llm.response_received | LLM响应接收 |
| | llm.stream_chunk | 流式分片到达 |
| 工具事件 | tool.call_started | 工具调用开始 |
| | tool.call_completed | 工具调用完成 |
| | tool.call_failed | 工具调用失败 |
| 任务事件 | task.created | 任务创建成功 |
| | task.executing | 任务开始执行 |
| | task.completed | 任务完成 |
| | task.failed | 任务失败 |
| 配置事件 | config.updated | 配置更新成功 |
| UI事件 | ui.state_updated | UI状态更新 |

### 5.2 事件处理流程

```
事件发布者                  事件总线                  事件订阅者
        |                      |                      |
        |--- publish(event) --->|                      |
        |                      |                      |
        |                      |--- 缓存事件 ---------->|
        |                      |                      |
        |                      |--- 持久化事件 -------->|
        |                      |                      |
        |                      |--- 通知订阅者 -------->|
        |                      |                      |
        |                      |<--- 处理完成 ----------|
        |                      |                      |
```

## 7. 容错与恢复

### 7.1 层级容错策略

| 层级 | 容错策略 |
|-----|---------|
| L1 | 持久化容错、缓存机制、模拟模式 |
| L2 | 事务操作、错误重试、状态恢复 |
| L3 | 任务重试、断点续执行、场景降级 |
| L4 | 请求重试、超时处理、负载均衡 |

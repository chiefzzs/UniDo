## 2. L1 基础设施层文件详解

### 2.1 L1b 持久化与IO

| 文件 | 职责 |
|------|------|
| `persistence_service.py` | 持久化服务核心，实现save/load/list/delete/exists接口 |
| `file_storage.py` | 文件存储实现，处理JSON文件读写 |
| `storage_factory.py` | 存储工厂，创建不同类型存储实例 |

**核心接口**：
```python
class PersistenceService:
    def save(self, entity_type: str, data: dict) -> str
    def load(self, entity_type: str, entity_id: str) -> Optional[dict]
    def list(self, entity_type: str, filters: dict = None) -> List[dict]
    def delete(self, entity_type: str, entity_id: str) -> bool
    def exists(self, entity_type: str, entity_id: str) -> bool
```

**关键数据结构**：

| 数据结构 | 字段 | 说明 |
|---------|------|------|
| **存储配置** | `base_path: str` | 存储根目录 |
| | `env: str` | 环境标识(dev/test) |
| | `file_extension: str` | 文件扩展名(.json) |
| **实体记录** | `entity_id: str` | 实体唯一标识 |
| | `created_at: str` | 创建时间(ISO 8601) |
| | `updated_at: str` | 更新时间(ISO 8601) |
| | `data: dict` | 业务数据 |

### 2.2 L1c 大模型基础设施

| 文件 | 职责 |
|------|------|
| `llm_client.py` | LLM客户端主入口，提供execute/execute_stream方法 |
| `api_adapters/base_adapter.py` | API适配器基类，定义统一接口 |
| `api_adapters/qwen_adapter.py` | Qwen模型API适配器 |
| `api_adapters/openai_adapter.py` | OpenAI API适配器 |
| `api_adapters/anthropic_adapter.py` | Anthropic(Claude) API适配器 |
| `request_constructor.py` | 请求构造器，构建LLM请求参数 |
| `response_parser.py` | 响应解析器，解析LLM返回数据 |

**核心接口**：
```python
class LLMClient:
    def execute(self, request: LLMRequest) -> LLMResponse
    def execute_stream(self, request: LLMRequest, on_chunk: Callable) -> LLMResponse
```

**关键数据结构**：

| 数据结构 | 字段 | 说明 |
|---------|------|------|
| **LLMRequest** | `model_name: str` | 模型名称 |
| | `messages: List[Dict]` | 消息列表 |
| | `tools: List[Dict]` | 工具描述列表(JSON Schema) |
| | `tool_choice: str` | 工具选择策略(auto/none/特定工具) |
| | `temperature: float` | 温度参数(0-1) |
| | `max_tokens: int` | 最大token数 |
| | `stream: bool` | 是否流式 |
| | `api_type: str` | API类型(qwen/openai/anthropic) |
| | `api_address: str` | API地址 |
| | `api_key: str` | API密钥 |
| **LLMResponse** | `content: str` | 响应内容 |
| | `finish_reason: str` | 结束原因(stop/length/tool_call) |
| | `model_name: str` | 实际模型名 |
| | `usage: Dict[str, int]` | token使用量 |
| **StreamChunk** | `chunk_id: str` | 分片ID |
| | `delta: str` | 分片内容 |
| | `chunk_type: str` | 分片类型(text/think/tool_call) |
| | `index: int` | 分片索引 |
| **LLMCallRecord** | `call_id: str` | 调用ID |
| | `session_id: str` | 会话ID |
| | `request: Dict` | 请求对象 |
| | `response: Dict` | 响应对象 |
| | `status: str` | 状态(completed/failed/in_progress) |
| | `duration_ms: int` | 耗时(毫秒) |

### 2.3 L1d 事件管理

| 文件 | 职责 |
|------|------|
| `event_bus.py` | 事件总线核心，提供publish/subscribe/get_events方法 |
| `event_types.py` | 事件类型常量定义 |
| `event_record.py` | 事件记录数据结构和转换 |
| `subscription.py` | 订阅管理，处理订阅和取消订阅 |

**核心接口**：
```python
class EventBus:
    def publish(self, event: Event, correlation_id: str = None) -> str
    def subscribe(self, event_type: str, callback: Callable) -> str
    def unsubscribe(self, subscription_id: str) -> bool
    def get_events_by_correlation(self, correlation_id: str) -> List[EventRecord]
```

**关键数据结构**：

| 数据结构 | 字段 | 说明 |
|---------|------|------|
| **Event** | `event_id: str` | 事件唯一标识 |
| | `event_type: str` | 事件类型 |
| | `timestamp: datetime` | 事件时间戳 |
| | `payload: Dict[str, Any]` | 事件载荷 |
| | `source_component: str` | 发起组件(L1/L2/L3/L4/L5) |
| | `source_service: str` | 发起服务 |
| **EventRecord** | `record_id: str` | 记录唯一标识 |
| | `event_id: str` | 关联事件ID |
| | `stored_at: str` | 存储时间 |
| | `correlation_id: str` | 关联ID(用于链路追踪) |
| **事件类型常量** | `project.created` | 项目创建 |
| | `session.created` | 会话创建 |
| | `message.created` | 消息创建 |
| | `llm.request_sent` | LLM请求发送 |
| | `llm.response_received` | LLM响应接收 |
| | `llm.stream_chunk` | 流式分片 |
| | `tool.execution_started` | 工具执行开始 |
| | `tool.execution_completed` | 工具执行完成 |

## 8. 数据流与层级调用关系

### 8.1 整体调用链

```
L5 UI组件  [src/static]
    ↓ (用户交互)
L4 网关层 (HTTP/WebSocket)  [services/L4_gateway]
    ↓ (API调用)
L3 场景协同层  [services/L3_scenario_coordination]
    ├── L3a: DialogueService → BaseExecutionService → [IntentService/ToolTaskExecutor/TaskGroupExecutor]
    └── L3c: UI操作组件
    ↓
L2 领域层  [services/L2_domain]
    ├── L2a: 领域实体管理
    ├── L2b: 记忆与状态管理
    ├── L2c: 工具执行服务
    ├── L2d: LLM执行服务
    ├── L2e: 请求构造服务
    ├── L2f: 工具管理服务
    └── L2g: 技能管理服务
    ↓
L1 基础设施层  [services/L1_infrastructure]
    ├── L1b: 持久化存储
    ├── L1c: LLM API调用
    └── L1d: 事件总线
```

### 8.2 对话处理流程

```
用户输入 → L5 ChatInterface
    → L4 HTTP API (POST /api/sessions/{id}/messages)
    → L3a DialogueService.process_dialogue()
    → L3a BaseExecutionService.execute_task()
    → L3a IntentService.analyze_intent()
    → L2d LLM执行服务
    → L2e 请求构造服务
    → L1c LLM API调用
    → L2d 解析工具调用
    → L2c 工具执行
    → L3a CheckTaskService / AdjustTaskService (递归)
    → L1d 发布事件
    → L4 WebSocket 推送
    → L5 展示结果
```

### 8.3 持久化流程

```
业务层调用 save()
    → L1b PersistenceService.save()
    → FileStorage.write()
    → JSON文件 (data/{env}/xxx.json)
```

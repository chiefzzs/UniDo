## 二、核心概念详细定义

### 2.1 配置层概念

#### 📦 Concept 1: Project（项目）

```python
@dataclass
class Project:
    """
    【Aggregate Root】⭐ 配置层核心
    领域定义：项目的根实体，是资源管理的最顶层容器
    生命周期：创建 → 配置 → 使用 → 关闭/删除
    职责：管理项目元数据、工作区配置、模型配置、会话列表
    """
    
    # === 标识与属性 ===
    project_id: str                    # 项目唯一标识 (UUID)
    name: str                          # 项目名称
    description: str                   # 项目描述（可选）
    created_at: datetime               # 创建时间
    updated_at: datetime               # 最后修改时间
    
    # === 关联配置 ===
    workspace: Workspace               # 工作区（实体）
    model_config: ModelConfig          # 模型配置（值对象）
    
    # === 关联实体 ===
    sessions: List[Session]           # 属于该项目的会话列表
    current_session_id: Optional[str]  # 当前打开的会话ID
    
    # === 状态 ===
    status: ProjectStatus
    
    class ProjectStatus(Enum):
        ACTIVE = "active"            # 活跃中
        ARCHIVED = "archived"         # 已归档
        DELETED = "deleted"           # 已删除
    
    # === 核心方法 ===
    def create_project(name: str, workspace_path: Path) -> 'Project':
        """创建新项目，初始化工作区和模型配置"""
    
    def open_session(session_id: str) -> Session:
        """打开指定会话"""
    
    def create_session() -> Session:
        """创建新会话"""
    
    def get_or_create_default_project() -> 'Project':
        """获取或创建缺省项目"""
```

#### 📦 Concept 2: Workspace（工作区）

```python
@dataclass
class Workspace:
    """
    【Entity】⭐ 配置层
    领域定义：工作区，管理文件目录结构
    生命周期：关联到 Project 后激活，可独立管理
    职责：管理根路径、文件树结构、当前打开文件
    """
    
    # === 标识 ===
    workspace_id: str                   # 工作区唯一标识
    project_id: str                    # 所属项目ID
    
    # === 文件系统 ===
    root_path: Path                    # 工作区根路径
    file_tree: FileTree                # 文件树结构
    current_file: Optional[Path]       # 当前打开的文件
    
    # === 快捷操作 ===
    recent_files: List[Path]           # 最近打开的文件
    bookmarks: List[Path]             # 收藏的文件/目录
    
    class FileTree:
        """文件树结构"""
        root: FileNode
        
        class FileNode:
            name: str
            path: Path
            type: FileType  # FILE / DIRECTORY
            children: List['FileNode']
            is_hidden: bool
            permissions: Permissions
    
    # === 核心方法 ===
    def get_file_tree() -> FileTree:
        """获取文件树"""
    
    def open_file(path: Path) -> FileContent:
        """打开文件"""
    
    def create_file(path: Path, content: str):
        """创建文件"""
    
    def get_default_workspace() -> 'Workspace':
        """获取缺省工作区，关联到本工具根目录"""
```

#### 📦 Concept 3: ModelConfig（模型配置）

```python
@dataclass
class ModelConfig:
    """
    【Value Object】⭐ 配置层
    领域定义：模型配置参数
    不可变性：创建后不可更改，如需修改需创建新实例
    职责：封装LLM调用的所有配置参数
    """
    
    # === 模型配置 ===
    model_name: str                    # 模型名称
    provider: str                     # 模型提供商
    
    # === 生成参数 ===
    temperature: float                # 温度参数 (0.0-2.0)
    max_tokens: int                   # 最大生成token数
    top_p: float                       # Top-p采样
    top_k: int                        # Top-k采样
    
    # === 系统提示 ===
    system_prompt: str                # 系统提示词
    
    # === 流式控制 ===
    stream: bool = True              # 是否启用流式响应
    
    # === 上下文控制 ===
    context_window: int               # 上下文窗口大小
    truncation_strategy: str          # 截断策略
    
    # === 工具列表 ===
    tools: List[ToolDefinition]      # 可用工具定义列表
    
    # === 核心方法 ===
    @classmethod
    def get_default_config() -> 'ModelConfig':
        """获取缺省模型配置"""
        return cls(
            model_name="Qwen/Qwen3.5-397B-A17B",
            provider="modelscope",
            temperature=0.7,
            max_tokens=16000,
            system_prompt="You are a powerful code assistant...",
            stream=True,
            tools=[]
        )

@dataclass
class ToolDefinition:
    """
    【Value Object】
    领域定义：工具定义，用于LLM调用
    """
    
    name: str                         # 工具名称
    description: str                  # 工具描述
    parameters: dict                  # 参数定义 (JSON Schema)
    required: List[str]               # 必需参数列表
```

### 2.2 会话层概念

#### 📦 Concept 4: Session（会话）

```python
@dataclass
class Session:
    """
    【Aggregate Root】⭐ 会话层核心
    领域定义：对话会话，管理完整的对话历史和上下文
    生命周期：创建 → 用户交互 → 持久化 → 关闭/删除
    职责：管理消息历史、关联任务计划、维护会话状态
    """
    
    # === 标识 ===
    session_id: str                   # 会话唯一标识 (UUID)
    project_id: str                    # 所属项目ID
    
    # === 消息历史 ===
    messages: List[Message]           # 消息历史列表（按时间顺序）
    
    # === 任务计划 ===
    task_plan: Optional[TaskPlan]     # 关联的任务计划（可选）
    
    # === 状态 ===
    status: SessionStatus
    created_at: datetime
    updated_at: datetime
    
    class SessionStatus(Enum):
        ACTIVE = "active"            # 活跃中
        IDLE = "idle"                 # 空闲
        PAUSED = "paused"             # 已暂停
        CLOSED = "closed"             # 已关闭
    
    # === 核心方法 ===
    def add_message(message: Message) -> None:
        """追加消息到历史"""
    
    def get_messages() -> List[Message]:
        """获取完整消息历史"""
    
    def create_task_plan() -> TaskPlan:
        """创建任务计划"""
    
    def clear_messages() -> None:
        """清空消息历史"""
```

#### 📦 Concept 5: Message（消息）

```python
@dataclass
class Message:
    """
    【Entity】
    领域定义：对话中的基本原子单元，代表一条具有角色的内容
    生命周期：创建 → 发送 → 接收 → 处理 → 持久化
    职责：承载对话内容、工具调用、思考过程等信息
    """
    
    # === 标识与属性 ===
    message_id: str                   # 消息唯一标识
    role: Role                        # 消息角色
    message_type: MessageType         # 消息类型
    content: Union[str, List[ContentItem]]  # 消息内容
    timestamp: datetime               # 时间戳
    
    # === 状态管理 ===
    status: MessageStatus             # 消息状态
    status_reason: Optional[str]      # 状态变更原因
    
    # === 工具调用 ===
    tool_calls: Optional[List[ToolCall]]  # 关联的工具调用（可选）
    tool_result: Optional[ToolResult]    # 工具执行结果（可选）
    
    # === 思考内容 ===
    reasoning_content: Optional[str]  # 思考过程（可选）
    
    # === 元数据 ===
    metadata: Dict[str, Any]          # 扩展元数据
    parent_message_id: Optional[str]  # 父消息ID（用于消息引用）
    conversation_id: Optional[str]    # 所属对话ID
    
    # === 枚举定义 ===
    class Role(Enum):
        """消息角色"""
        SYSTEM = "system"            # 系统消息 - 系统指令、提示词等
        USER = "user"                # 用户消息 - 用户输入的请求
        ASSISTANT = "assistant"      # 助手消息 - AI助手的响应
        TOOL_RESULT = "tool_result"  # 工具结果消息 - 工具执行的结果
    
    class MessageType(Enum):
        """消息类型 - 更细致的分类"""
        # 文本类型
        TEXT = "text"                # 纯文本消息
        REASONING = "reasoning"      # 思考内容消息
        SUMMARY = "summary"          # 摘要消息
        
        # 工具相关类型
        TOOL_CALL = "tool_call"      # 工具调用消息
        TOOL_RESPONSE = "tool_response"  # 工具响应消息
        
        # 任务相关类型
        TASK_CREATION = "task_creation"  # 任务创建消息
        TASK_UPDATE = "task_update"      # 任务更新消息
        TASK_COMPLETION = "task_completion"  # 任务完成消息
        
        # 文件相关类型
        FILE_ATTACHMENT = "file_attachment"  # 文件附件消息
        CODE_BLOCK = "code_block"      # 代码块消息
        DIFF = "diff"                # 差异对比消息
        
        # 多模态类型
        MULTIMODAL = "multimodal"    # 多模态混合消息
        IMAGE = "image"              # 图片消息
        AUDIO = "audio"              # 音频消息
        
        # 系统类型
        ERROR = "error"              # 错误消息
        WARNING = "warning"          # 警告消息
        INFO = "info"                # 信息提示消息
        SYSTEM_NOTIFICATION = "system_notification"  # 系统通知消息
    
    class MessageStatus(Enum):
        """消息状态 - 完整的状态生命周期"""
        # 创建状态
        CREATED = "created"          # 已创建 - 消息刚被创建，尚未发送
        PENDING = "pending"          # 等待中 - 等待发送或处理
        
        # 传输状态
        SENDING = "sending"          # 发送中 - 正在发送
        SENT = "sent"                # 已发送 - 已成功发送
        DELIVERED = "delivered"      # 已送达 - 已成功送达接收方
        
        # 处理状态
        PROCESSING = "processing"    # 处理中 - 正在被处理
        RECEIVED = "received"        # 已接收 - 已被接收方接收
        READ = "read"                # 已读 - 已被用户阅读
        
        # 完成状态
        COMPLETED = "completed"      # 已完成 - 处理完成
        ACKNOWLEDGED = "acknowledged"  # 已确认 - 已被确认
        
        # 异常状态
        FAILED = "failed"            # 失败 - 发送或处理失败
        CANCELLED = "cancelled"      # 已取消 - 已被取消
        TIMEOUT = "timeout"          # 超时 - 发送或处理超时
        RETRYING = "retrying"        # 重试中 - 正在重试
        
        # 特殊状态
        STREAMING = "streaming"      # 流式传输中 - 正在流式传输
        STREAM_COMPLETE = "stream_complete"  # 流式传输完成
    
    class ContentItem:
        """多模态内容项"""
        type: str  # text/image/file/code/audio
        text: Optional[str]
        image_url: Optional[str]
        file_path: Optional[Path]
        code: Optional[str]
        language: Optional[str]
        audio_url: Optional[str]
        metadata: Dict[str, Any]
    
    # === 核心方法 ===
    @classmethod
    def create_system_message(content: str, system_prompt: Optional[str] = None) -> 'Message':
        """创建系统消息"""
        return cls(
            message_id=str(uuid.uuid4()),
            role=Message.Role.SYSTEM,
            message_type=Message.MessageType.TEXT,
            content=system_prompt or content,
            timestamp=datetime.now(),
            status=Message.MessageStatus.CREATED
        )
    
    @classmethod
    def create_user_message(content: str, attachments: Optional[List[Path]] = None) -> 'Message':
        """创建用户消息"""
        return cls(
            message_id=str(uuid.uuid4()),
            role=Message.Role.USER,
            message_type=Message.MessageType.TEXT,
            content=content,
            timestamp=datetime.now(),
            status=Message.MessageStatus.CREATED
        )
    
    @classmethod
    def create_assistant_message(content: str, reasoning_content: Optional[str] = None) -> 'Message':
        """创建助手消息"""
        return cls(
            message_id=str(uuid.uuid4()),
            role=Message.Role.ASSISTANT,
            message_type=Message.MessageType.TEXT,
            content=content,
            reasoning_content=reasoning_content,
            timestamp=datetime.now(),
            status=Message.MessageStatus.CREATED
        )
    
    @classmethod
    def create_tool_call_message(tool_calls: List[ToolCall]) -> 'Message':
        """创建工具调用消息"""
        return cls(
            message_id=str(uuid.uuid4()),
            role=Message.Role.ASSISTANT,
            message_type=Message.MessageType.TOOL_CALL,
            content="",
            tool_calls=tool_calls,
            timestamp=datetime.now(),
            status=Message.MessageStatus.CREATED
        )
    
    @classmethod
    def create_tool_result_message(tool_result: ToolResult, tool_call_id: str) -> 'Message':
        """创建工具结果消息"""
        return cls(
            message_id=str(uuid.uuid4()),
            role=Message.Role.TOOL_RESULT,
            message_type=Message.MessageType.TOOL_RESPONSE,
            content=str(tool_result.data) if tool_result.data else "",
            tool_result=tool_result,
            timestamp=datetime.now(),
            status=Message.MessageStatus.CREATED,
            metadata={"tool_call_id": tool_call_id}
        )
    
    @classmethod
    def create_error_message(error_message: str, error_type: Optional[str] = None) -> 'Message':
        """创建错误消息"""
        return cls(
            message_id=str(uuid.uuid4()),
            role=Message.Role.SYSTEM,
            message_type=Message.MessageType.ERROR,
            content=error_message,
            timestamp=datetime.now(),
            status=Message.MessageStatus.CREATED,
            metadata={"error_type": error_type}
        )
    
    def update_status(self, new_status: 'MessageStatus', reason: Optional[str] = None) -> None:
        """更新消息状态"""
        self.status = new_status
        self.status_reason = reason
    
    def is_streaming(self) -> bool:
        """判断是否正在流式传输"""
        return self.status == Message.MessageStatus.STREAMING
    
    def is_failed(self) -> bool:
        """判断是否失败"""
        return self.status in [Message.MessageStatus.FAILED, Message.MessageStatus.TIMEOUT]
    
    def can_retry(self) -> bool:
        """判断是否可以重试"""
        return self.status in [Message.MessageStatus.FAILED, Message.MessageStatus.TIMEOUT]
    
    def get_text_content(self) -> Optional[str]:
        """获取文本内容"""
        if isinstance(self.content, str):
            return self.content
        elif isinstance(self.content, list):
            text_items = [item.text for item in self.content if item.type == 'text' and item.text]
            return '\n'.join(text_items) if text_items else None
        return None
```

###### 1.1 任务执行流程详细说明

任务执行分为四个核心阶段：

| 阶段 | 主要工作 | 创建的消息/记录 |
|------|----------|----------------|
| **阶段1: 计划任务** | 分析任务意图、理解目标、选择工具、制定计划 | TASK_UPDATE、TaskAnalysis |
| **阶段2: 执行任务** | 执行工具调用（可能多次循环）| TOOL_CALL、TOOL_RESPONSE、TaskToolExecution、TASK_UPDATE |
| **阶段3: 检查任务结果** | 验证执行结果、检查是否满足目标、分析问题 | TaskToolResultAnalysis、TASK_UPDATE |
| **阶段4: 调整任务** | 根据检查结果决策：完成/重试/重新计划 | TASK_COMPLETION 或 TASK_UPDATE |

###### 1.2 任务组执行流程

当需要执行多个任务时，使用任务组（TaskGroup）：

```
[创建 TaskGroup] - 任务组容器
    ↓
    ┌─────────────────────────────────────────┐
    │ 任务组执行循环                          │
    └─────────────────────────────────────────┘
    ↓
    ┌─────────────────────────────────────────┐
    │ 获取下一个待执行任务                     │
    └─────────────────────────────────────────┘
    ↓
    执行单个任务（走上述"任务执行完整流程"）
    ↓
    ┌─────────────────────────────────────────┐
    │ 任务组内还有未执行任务？                 │
    └─────────────────────────────────────────┘
    ↓ 是                    ↓ 否
继续循环             任务组完成
    ↓
[创建 TaskGroup 聚合结果]
    ↓
[创建 SUMMARY 消息] - 任务组总结
```

任务组支持两种执行模式：
1. **顺序执行** - 任务按顺序依次执行，前一个完成后开始下一个
2. **并行执行** - 多个任务可以同时执行（适用于无依赖的任务）
3. **依赖执行** - 根据任务间的依赖关系确定执行顺序

###### 3. 实际会话流程示例

```python
@dataclass
class SessionMessageManager:
    """会话消息管理器 - 负责各类消息的创建和读取"""
    
    session: Session
    
    # === 消息创建方法 ===
    
    def add_system_prompt(self, prompt: str) -> Message:
        """创建并添加系统提示词消息"""
        msg = Message.create_system_message(prompt)
        msg.update_status(Message.MessageStatus.PENDING)
        self.session.add_message(msg)
        msg.update_status(Message.MessageStatus.COMPLETED)
        return msg
    
    def add_user_message(self, content: str, 
                        attachments: Optional[List[Path]] = None) -> Message:
        """创建并添加用户消息"""
        msg = Message.create_user_message(content, attachments)
        msg.update_status(Message.MessageStatus.SENDING)
        self.session.add_message(msg)
        msg.update_status(Message.MessageStatus.DELIVERED)
        return msg
    
    def start_assistant_response(self, initial_content: str = "",
                                reasoning_content: Optional[str] = None) -> Message:
        """开始助手流式响应"""
        msg = Message.create_assistant_message(initial_content, reasoning_content)
        msg.message_type = Message.MessageType.TEXT
        msg.update_status(Message.MessageStatus.STREAMING)
        self.session.add_message(msg)
        return msg
    
    def append_to_stream(self, message: Message, new_content: str) -> None:
        """追加内容到流式消息"""
        if isinstance(message.content, str):
            message.content += new_content
        else:
            # 处理多模态内容
            pass
    
    def complete_stream(self, message: Message) -> None:
        """完成流式响应"""
        message.update_status(
            Message.MessageStatus.STREAM_COMPLETE,
            reason="流式响应完成"
        )
    
    def add_tool_call(self, tool_calls: List[ToolCall]) -> Message:
        """创建并添加工具调用消息"""
        msg = Message.create_tool_call_message(tool_calls)
        msg.update_status(Message.MessageStatus.PENDING)
        self.session.add_message(msg)
        msg.update_status(Message.MessageStatus.PROCESSING)
        return msg
    
    def add_tool_result(self, tool_result: ToolResult, 
                       tool_call_id: str) -> Message:
        """创建并添加工具结果消息"""
        msg = Message.create_tool_result_message(tool_result, tool_call_id)
        msg.update_status(Message.MessageStatus.RECEIVED)
        self.session.add_message(msg)
        msg.update_status(Message.MessageStatus.COMPLETED)
        return msg
    
    def add_task_creation(self, task: Task) -> Message:
        """创建并添加任务创建通知消息"""
        msg = Message(
            message_id=str(uuid.uuid4()),
            role=Message.Role.SYSTEM,
            message_type=Message.MessageType.TASK_CREATION,
            content=f"创建任务: {task.description}",
            timestamp=datetime.now(),
            status=Message.MessageStatus.CREATED,
            metadata={"task_id": task.task_id}
        )
        self.session.add_message(msg)
        msg.update_status(Message.MessageStatus.COMPLETED)
        return msg
    
    def add_task_update(self, task: Task, update_info: str) -> Message:
        """创建并添加任务更新消息"""
        msg = Message(
            message_id=str(uuid.uuid4()),
            role=Message.Role.SYSTEM,
            message_type=Message.MessageType.TASK_UPDATE,
            content=f"任务更新: {update_info}",
            timestamp=datetime.now(),
            status=Message.MessageStatus.CREATED,
            metadata={"task_id": task.task_id, "new_status": task.status.value}
        )
        self.session.add_message(msg)
        msg.update_status(Message.MessageStatus.COMPLETED)
        return msg
    
    def add_task_completion(self, task: Task, 
                          success: bool, result_summary: str) -> Message:
        """创建并添加任务完成消息"""
        msg = Message(
            message_id=str(uuid.uuid4()),
            role=Message.Role.SYSTEM,
            message_type=Message.MessageType.TASK_COMPLETION,
            content=f"任务{'成功' if success else '失败'}: {result_summary}",
            timestamp=datetime.now(),
            status=Message.MessageStatus.CREATED,
            metadata={"task_id": task.task_id, "success": success}

                current = parent
```

```python
@dataclass
class TaskExecutor:
    """任务执行器 - 实现四阶段任务执行流程"""
    
    msg_manager: SessionMessageManager
    
    def execute_task(self, task: Task) -> Task:
        """执行完整的任务流程"""
        print(f"\n开始执行任务: {task.description}")
        
        # ========== 阶段1: 计划任务 ==========
        task = self.plan_task(task)
        if task.status == Task.TaskStatus.FAILED:
            return task
            
        # ========== 阶段2: 执行任务 ==========
        task = self.execute_task_tools(task)
        if task.status == Task.TaskStatus.FAILED:
            return task
            
        # 循环：检查 → 调整 → 可能重新执行
        max_retries = 3
        retry_count = 0
        
        while retry_count < max_retries:
            # ========== 阶段3: 检查任务结果 ==========
            task, is_complete = self.check_task_result(task)
            if is_complete:
                break
                
            # ========== 阶段4: 调整任务 ==========
            task, retry = self.adjust_task(task)
            if not retry:
                break
                
            retry_count += 1
            print(f"重试任务 (第 {retry_count}/{max_retries} 次)")
        
        # 任务完成
        task.status = Task.TaskStatus.COMPLETED
        self.msg_manager.add_task_completion(task, True, "任务执行完成")
        
        return task
    
    def plan_task(self, task: Task) -> Task:
        """阶段1: 计划任务"""
        print("  → 阶段1: 计划任务")
        
        # 更新任务状态
        task.status = Task.TaskStatus.ANALYZING
        self.msg_manager.add_task_update(task, "开始任务分析")
        
        # 模拟任务分析
        task_analysis = TaskAnalysis(
            analysis_id=f"analysis_{task.task_id}",
            task_id=task.task_id,
            goal_clarification=f"目标: {task.description}",
            execution_strategy="使用Read工具读取文件，然后分析",
            required_tools=["Read", "SearchCodebase"],
            tool_selection_reason="需要读取和搜索代码",
            expected_output="代码分析结果",
            risks=["文件不存在", "代码结构复杂"],
            mitigation_plan="遇到问题会尝试其他方法",
            validation_status=TaskAnalysis.ValidationStatus.PENDING,
            validation_errors=[]
        )
        
        # 保存分析结果
        task.task_analysis = task_analysis
        
        # 更新状态
        self.msg_manager.add_task_update(task, "计划完成，准备执行")
        task.status = Task.TaskStatus.READY
        
        return task
    
    def execute_task_tools(self, task: Task) -> Task:
        """阶段2: 执行任务工具"""
        print("  → 阶段2: 执行任务")
        
        task.status = Task.TaskStatus.EXECUTING
        
        # 模拟执行工具（这里假设有两个工具需要执行）
        tools_to_execute = [
            ("Read", {"file_path": "/workspace/main.py"}),
            ("SearchCodebase", {"query": "class definitions"})
        ]
        
        for tool_name, params in tools_to_execute:
            # 创建工具调用消息
            tool_call = ToolCall(
                tool_call_id=f"call_{task.task_id}_{tool_name}",
                tool_name=tool_name,
                parameters=params
            )
            tool_call_msg = self.msg_manager.add_tool_call([tool_call])
            
            # 记录工具执行
            execution = TaskToolExecution(
                execution_id=f"exec_{tool_call.tool_call_id}",
                task_id=task.task_id,
                tool_name=tool_name,
                parameters=params,
                status=TaskToolExecution.ToolExecutionStatus.PENDING,
                retry_count=0,
                max_retries=3
            )
            
            if not task.tool_executions:
                task.tool_executions = []
            task.tool_executions.append(execution)
            
            # 执行工具（模拟）
            execution.status = TaskToolExecution.ToolExecutionStatus.RUNNING
            execution.started_at = datetime.now()
            
            # 模拟工具执行结果
            tool_result = ToolResult(
                result_id=f"result_{execution.execution_id}",
                tool_name=tool_name,
                success=True,
                data=f"执行{tool_name}的结果",
                output_type=ToolResult.OutputType.TEXT,
                metadata=params,
                generated_at=datetime.now()
            )
            
            execution.result = tool_result
            execution.completed_at = datetime.now()
            execution.execution_duration = 1.5
            execution.status = TaskToolExecution.ToolExecutionStatus.COMPLETED
            
            # 创建工具结果消息
            self.msg_manager.add_tool_result(tool_result, tool_call.tool_call_id)
            self.msg_manager.add_task_update(task, f"工具 {tool_name} 执行完成")
            
        return task
    
    def check_task_result(self, task: Task) -> Tuple[Task, bool]:
        """阶段3: 检查任务结果"""
        print("  → 阶段3: 检查任务结果")
        
        self.msg_manager.add_task_update(task, "开始检查执行结果")
        
        # 创建结果分析
        result_analysis = TaskToolResultAnalysis(
            analysis_id=f"result_analysis_{task.task_id}",
            task_id=task.task_id,
            task_tool_execution_id=task.tool_executions[-1].execution_id if task.tool_executions else "",
            tool_result=task.tool_executions[-1].result if task.tool_executions else None,
            analysis_report="结果分析：看起来不错",
            summary="任务执行成功",
            key_findings=["发现了重要信息", "满足任务目标"],
            validation_status=TaskToolResultAnalysis.ValidationStatus.PASSED,
            validation_criteria=["工具执行成功", "结果有意义"],
            validation_results={"工具执行成功": True, "结果有意义": True},
            validation_errors=[],
            next_action=TaskToolResultAnalysis.NextAction.COMPLETE,
            action_reason="任务已完成"
        )
        
        task.result_analysis = result_analysis
        self.msg_manager.add_task_update(task, "检查完成")
        
        # 判断是否完成
        is_complete = (result_analysis.next_action == TaskToolResultAnalysis.NextAction.COMPLETE)
        
        return task, is_complete
    
    def adjust_task(self, task: Task) -> Tuple[Task, bool]:
        """阶段4: 调整任务"""
        print("  → 阶段4: 调整任务")
        
        analysis = task.result_analysis
        
        if analysis.next_action == TaskToolResultAnalysis.NextAction.RETRY:
            # 重试某些工具
            self.msg_manager.add_task_update(task, "重试中")
            return task, True  # 继续重试
        elif analysis.next_action == TaskToolResultAnalysis.NextAction.REPLAN:
            # 重新计划任务
            self.msg_manager.add_task_update(task, "重新计划中")
            return task, True  # 返回计划阶段
        else:
            # 其他情况，完成任务
            return task, False
```

```python
@dataclass
class MessagePersistence:
    """消息持久化管理器"""
    
    storage_path: Path
    
    def save_messages(self, messages: List[Message]) -> None:
        """保存消息到存储"""
        data = []
        for msg in messages:
            msg_data = {
                "message_id": msg.message_id,
                "role": msg.role.value,
                "message_type": msg.message_type.value,
                "content": self._serialize_content(msg.content),
                "timestamp": msg.timestamp.isoformat(),
                "status": msg.status.value,
                "status_reason": msg.status_reason,
                "metadata": msg.metadata,
                "parent_message_id": msg.parent_message_id,
                "conversation_id": msg.conversation_id,
                "tool_calls": self._serialize_tool_calls(msg.tool_calls),
                "tool_result": self._serialize_tool_result(msg.tool_result),
                "reasoning_content": msg.reasoning_content
            }
            data.append(msg_data)
        
        with open(self.storage_path / "messages.json", "w", encoding="utf-8") as f:
            json.dump(data, f, ensure_ascii=False, indent=2)
    
    def load_messages(self) -> List[Message]:
        """从存储加载消息"""
        file_path = self.storage_path / "messages.json"

            return []

            data = json.load(f)
        
        messages = []
        for msg_data in data:
            msg = self._deserialize_message(msg_data)
            messages.append(msg)
        
        return messages
```

## 五、概念验证规则

### 5.1 Project 验证规则

```python
def validate_project(project: Project) -> Tuple[bool, List[str]]:
    """验证项目的有效性"""
    errors = []
    
    # 名称验证
    if not project.name or len(project.name.strip()) == 0:
        errors.append("项目名称不能为空")
    
    # 工作区验证
    if not project.workspace:
        errors.append("项目必须关联工作区")
    elif not project.workspace.root_path.exists():
        errors.append(f"工作区路径不存在: {project.workspace.root_path}")
    
    # 模型配置验证
    if not project.model_config:
        errors.append("项目必须关联模型配置")
    elif project.model_config.temperature < 0 or project.model_config.temperature > 2:
        errors.append("温度参数必须在0-2之间")
    
    return len(errors) == 0, errors
```

### 5.2 Task 验证规则

```python
def validate_task(task: Task) -> Tuple[bool, List[str]]:
    """验证任务的有效性"""
    errors = []
    
    # 描述验证
    if not task.description or len(task.description.strip()) == 0:
        errors.append("任务描述不能为空")
    
    # 优先级验证
    if task.priority < 1 or task.priority > 10:
        errors.append("优先级必须在1-10之间")
    
    # 输入数据验证
    if task.input_data is None:
        errors.append("任务必须有输入数据")
```

```python
def validate_intent_understanding(analysis: IntentUnderstanding) -> Tuple[bool, List[str]]:
    """验证意图分析的有效性"""
    errors = []
    
    # 置信度验证
    if analysis.confidence < 0 or analysis.confidence > 1:
        errors.append("置信度必须在0-1之间")
```

## 六、概念持久化策略

### 6.1 聚合根持久化

- **Project**: 作为顶级聚合根，独立持久化
- **Session**: 作为Project的子聚合，通过project_id关联
- **TaskPlan**: 作为Session的子聚合，通过session_id关联
- **TaskGroup**: 作为TaskPlan的子聚合，通过task_plan_id关联

### 6.2 实体持久化

- **Workspace**: 作为Project的内部实体，与Project一起持久化
- **Message**: 作为Session的内部实体，与Session一起持久化
- **Task**: 作为TaskGroup的内部实体，与TaskGroup一起持久化

### 6.3 值对象持久化

- **ModelConfig**: 作为Project的属性，序列化存储
- **ToolDefinition**: 作为ModelConfig的一部分，序列化存储
- **ToolResult**: 作为TaskToolExecution的属性，序列化存储
- **ToolError**: 作为TaskToolExecution的属性，序列化存储

## 九、任务概念模型详细流程

### 9.1 任务流程总览

任何输入的待处理信息都是任务的起点。任务流程通过递归方式处理各种复杂场景。

```
┌─────────────────────────────────────────────────────────────────┐
│                      任务流程总览                              │
└─────────────────────────────────────────────────────────────────┘

                          ┌──────────────┐
                          │  任务输入    │ ← 任何待处理的信息
                          └──────┬───────┘
                                 │
                                 ↓
                    ┌───────────────────────┐
                    │  1. 意图分析/任务计划  │
                    └───────────┬───────────┘
                                │
            ┌───────────────────┼───────────────────┐
            │                   │                   │
            ↓                   ↓                   ↓
    ┌───────────────┐  ┌────────────────┐  ┌─────────────────┐
    │  a) 直接完成  │  │ b) 单工具执行  │  │ c) 任务组执行   │
    │   任务        │  │   子流程       │  │   子流程        │
    └───────┬───────┘  └────────┬───────┘  └────────┬────────┘
            │                   │                   │
            └───────────────────┼───────────────────┘
                                ↓
                        ┌───────────────┐
                        │   任务完成    │
                        └───────────────┘
```

---

### 9.2 任务流程详细说明

#### 9.2.1 阶段1: 意图分析/任务计划

```
任务输入
    ↓
┌─────────────────────────────────────────────────┐
│ 意图分析                                        │
│ • 理解任务目标                                  │
│ • 分析输入信息                                  │
│ • 判断执行路径                                  │
└───────────────────┬─────────────────────────────┘
                    │
    ┌───────────────┼───────────────┐
    │               │               │
    ↓               ↓               ↓
   a)              b)              c)
直接完成        单工具执行       任务组执行
    ↓               ↓               ↓
    └───────────────┴───────────────┘
                    ↓
              任务完成
```

**意图分析的三个分支决策：**

| 分支 | 判定条件 | 后续流程 |
|------|----------|----------|
| **a) 直接完成任务** | 任务目标简单，无需调用工具或子任务 | 直接返回结果，任务结束 |
| **b) 单工具执行** | 需要调用单个工具来完成任务 | 转"单工具执行、检查、调整子流程" |
| **c) 任务组执行** | 需要多个任务协作完成，或任务复杂度高 | 转"任务组执行子流程" |

---

#### 9.2.2 分支b: 单工具执行、检查、调整子流程

```
┌───────────────────────────────────────────────────────────────┐
│              单工具执行子流程                                  │
└───────────────────────────────────────────────────────────────┘

      任务输入 (从意图分析传入)
              ↓
    ┌─────────────────────────┐
    │ 1. 工具执行              │ ← 选择并执行工具
    └───────────┬─────────────┘
                ↓
    ┌─────────────────────────┐
    │ 收集中间输出             │
    │ 收集结果输出             │
    │ (异常必须有明确异常信息)  │
    └───────────┬─────────────┘
                ↓
    ┌─────────────────────────┐
    │ 2. 检查任务              │ ← 递归执行任务流程
    │ 基于:                    │
    │ • 任务输入               │
    │ • 工具执行结果           │
    └───────────┬─────────────┘
                ↓
    ┌─────────────────────────┐
    │ 3. 调整任务              │ ← 递归执行任务流程
    │ 基于:                    │
    │ • 任务执行情况           │
    │ • 检查结果               │
    └───────────┬─────────────┘
                ↓
          此任务完成
```

**检查任务的递归处理：**
- 检查任务本身就是一个独立的任务
- 递归调用完整任务流程来执行检查
- 输入：原任务输入 + 工具执行结果
- 输出：检查结果（通过/不通过，问题分析）

**调整任务的递归处理：**
- 调整任务本身就是一个独立的任务
- 递归调用完整任务流程来执行调整
- 输入：任务执行情况 + 检查结果
- 输出：调整决策（完成/重试/重新计划）

---

#### 9.2.3 分支c: 任务组执行子流程

```
┌───────────────────────────────────────────────────────────────┐
│              任务组执行子流程                                  │
└───────────────────────────────────────────────────────────────┘

      任务输入 (从意图分析传入)
              ↓
    ┌─────────────────────────┐
    │ 创建任务组              │
    │ 规划子任务序列          │
    └───────────┬─────────────┘
                ↓
    ┌─────────────────────────┐
    │ 子任务执行循环          │
    └───────────┬─────────────┘
                ↓
    ┌─────────────────────────┐
    │ 获取下一个待执行子任务   │
    └───────────┬─────────────┘
                ↓
    ┌─────────────────────────┐
    │ 递归调用任务流程        │ ← 执行单个子任务
    │ (完整的任务流程)        │
    └───────────┬─────────────┘
                ↓
    ┌─────────────────────────┐
    │ 子任务完成              │
    │ 收集子任务结果          │
    └───────────┬─────────────┘
                ↓
    ┌─────────────────────────┐
    │ 还有未执行的子任务？     │
    └───────┬───────┬─────────┘
            ↓ 是    ↓ 否
        继续循环   任务组完成
            ↓
        聚合所有子任务结果
            ↓
      此任务组任务完成
```

**任务组执行特点：**
1. **递归性** - 每个子任务都递归调用完整任务流程
2. **顺序性** - 子任务按顺序依次执行（可支持并行/依赖模式）
3. **聚合性** - 最后聚合所有子任务的结果

```python
@dataclass
class RecursiveTaskExecutor:
    """
    递归任务执行器 - 实现完整的递归任务流程
    """
    
    msg_manager: 'SessionMessageManager'
    
    def execute_task(self, task: Task) -> Task:
        """
        执行完整的任务流程（递归入口）
        """
        print(f"开始执行任务: {task.task_id}")
        
        # 记录开始时间
        task.started_at = datetime.now()
        task.status = Task.TaskStatus.ANALYZING
        
        # ========== 阶段1: 意图分析/任务计划 ==========
        intent_result = self.perform_intent_analysis(task)
        
        # 根据意图分析结果选择执行路径
        if intent_result.execution_path == IntentAnalysisResult.ExecutionPath.DIRECT_COMPLETION:
            # 分支a: 直接完成
            task = self.execute_direct_completion(task, intent_result)
        elif intent_result.execution_path == IntentAnalysisResult.ExecutionPath.SINGLE_TOOL:
            # 分支b: 单工具执行
            task = self.execute_single_tool_task(task, intent_result)
        elif intent_result.execution_path == IntentAnalysisResult.ExecutionPath.TASK_GROUP:
            # 分支c: 任务组执行
            task = self.execute_task_group_task(task, intent_result)
        
        # 任务完成
        task.status = Task.TaskStatus.COMPLETED
        task.completed_at = datetime.now()
        
        return task
    
    def perform_intent_analysis(self, task: Task) -> IntentAnalysisResult:
        """
        执行意图分析
        """
        print(f"  → 意图分析: {task.task_id}")
        
        # 记录执行步骤
        step = TaskExecutionStep(
            step_id=f"step_{task.task_id}_intent",
            step_type=TaskExecutionStep.StepType.INTENT_ANALYSIS,
            step_name="意图分析",
            description="分析任务意图，选择执行路径",
            input_data=task.input_data,
            status=TaskExecutionStep.StepStatus.RUNNING,
            started_at=datetime.now()
        )
        task.execution_history.append(step)
        
        # 模拟意图分析（实际应该调用LLM）
        # 这里简化为：根据输入大小判断
        input_size = len(str(task.input_data))
        
        if input_size < 50:
            # 简单输入，直接完成
            execution_path = IntentAnalysisResult.ExecutionPath.DIRECT_COMPLETION
            intent_type = IntentAnalysisResult.IntentType.SIMPLE_QUERY
            direct_info = DirectCompletionInfo(
                result_summary="简单查询已处理",
                result_details={"answer": "这是直接回答"}
            )
            tool_info = None
            group_info = None
        elif input_size < 200:
            # 中等复杂度，单工具执行
            execution_path = IntentAnalysisResult.ExecutionPath.SINGLE_TOOL
            intent_type = IntentAnalysisResult.IntentType.TOOL_OPERATION
            direct_info = None
            tool_info = SingleToolInfo(
                tool_name="Read",
                tool_parameters={"file_path": "/workspace/test.txt"},
                expected_output="读取文件内容"
            )
            group_info = None
        else:
            # 高复杂度，任务组执行
            execution_path = IntentAnalysisResult.ExecutionPath.TASK_GROUP
            intent_type = IntentAnalysisResult.IntentType.COMPLEX_TASK
            direct_info = None
            tool_info = None
            group_info = TaskGroupInfo(
                subtasks=[
                    SubtaskDefinition(
                        subtask_id=f"{task.task_id}_sub1",
                        description="分析输入数据",
                        input_data={"data": task.input_data},
                        dependencies=[]
                    ),
                    SubtaskDefinition(
                        subtask_id=f"{task.task_id}_sub2",
                        description="处理分析结果",
                        input_data={"depends_on": "sub1"},
                        dependencies=[f"{task.task_id}_sub1"]
                    )
                ],
                execution_mode=TaskGroupInfo.ExecutionMode.SEQUENTIAL
            )
        
        # 完成步骤
        step.status = TaskExecutionStep.StepStatus.COMPLETED
        step.completed_at = datetime.now()
        step.output_data = {
            "execution_path": execution_path.value,
            "intent_type": intent_type.value
        }
        
        return IntentAnalysisResult(
            analysis_id=f"analysis_{task.task_id}",
            task_id=task.task_id,
            intent_type=intent_type,
            execution_path=execution_path,
            goal_clarification="理解任务目标",
            reasoning="基于输入分析选择路径",
            confidence=0.9,
            direct_completion_info=direct_info,
            single_tool_info=tool_info,
            task_group_info=group_info
        )
    
    def execute_direct_completion(self, task: Task, 
                                  intent_result: IntentAnalysisResult) -> Task:
        """
        分支a: 直接完成任务
        """
        print(f"  → 直接完成任务: {task.task_id}")
        
        step = TaskExecutionStep(
            step_id=f"step_{task.task_id}_direct",
            step_type=TaskExecutionStep.StepType.DIRECT_COMPLETION,
            step_name="直接完成",
            description="无需工具，直接完成任务",
            status=TaskExecutionStep.StepStatus.RUNNING,
            started_at=datetime.now()
        )
        task.execution_history.append(step)
        
        # 直接输出结果
        task.output_data = {
            "completion_type": "direct",
            "result": intent_result.direct_completion_info.result_details
        }
        
        step.status = TaskExecutionStep.StepStatus.COMPLETED
        step.completed_at = datetime.now()
        step.output_data = task.output_data
        
        # 添加消息
        self.msg_manager.add_task_completion(task, True, "直接完成任务")
        
        return task
    
    def execute_single_tool_task(self, task: Task,
                                 intent_result: IntentAnalysisResult) -> Task:
        """
        分支b: 单工具执行、检查、调整子流程
        """
        print(f"  → 单工具执行任务: {task.task_id}")
        
        max_iterations = 3
        iteration = 0
        
        while iteration < max_iterations:
            iteration += 1
            
            # ---------- 1. 工具执行 ----------
            task = self.execute_tool_in_task(task, intent_result.single_tool_info)
            
            if task.status == Task.TaskStatus.FAILED:
                break
            
            # ---------- 2. 检查任务（递归调用任务流程） ----------
            check_task = self.create_check_task(task)
            check_result = self.execute_task(check_task)  # 递归调用
            
            # 记录检查步骤
            check_step = TaskExecutionStep(
                step_id=f"step_{task.task_id}_check_{iteration}",
                step_type=TaskExecutionStep.StepType.TASK_CHECK,
                step_name="检查任务",
                description="递归检查任务执行结果",
                input_data={"original_task": task.task_id, "check_result": check_result.output_data},
                status=TaskExecutionStep.StepStatus.COMPLETED,
                started_at=datetime.now(),
                completed_at=datetime.now()
            )
            task.execution_history.append(check_step)
            
            # ---------- 3. 调整任务（递归调用任务流程） ----------
            adjust_task = self.create_adjustment_task(task, check_result)
            adjust_result = self.execute_task(adjust_task)  # 递归调用
            
            # 记录调整步骤
            adjust_step = TaskExecutionStep(
                step_id=f"step_{task.task_id}_adjust_{iteration}",
                step_type=TaskExecutionStep.StepType.TASK_ADJUSTMENT,
                step_name="调整任务",
                description="基于检查结果调整任务",
                input_data={"check_result": check_result.output_data, "adjust_result": adjust_result.output_data},
                status=TaskExecutionStep.StepStatus.COMPLETED,
                started_at=datetime.now(),
                completed_at=datetime.now()
            )
            task.execution_history.append(adjust_step)
            
            # 判断是否完成
            if adjust_result.output_data.get("action") == "complete":
                task.output_data = {"result": "成功", "iterations": iteration}
                break
        
        return task
    
    def execute_tool_in_task(self, task: Task, 
                             tool_info: SingleToolInfo) -> Task:
        """
        执行工具（单工具子流程的步骤1）
        """
        step = TaskExecutionStep(
            step_id=f"step_{task.task_id}_tool",
            step_type=TaskExecutionStep.StepType.TOOL_EXECUTION,
            step_name=f"工具执行: {tool_info.tool_name}",
            description="执行单个工具",
            input_data=tool_info.tool_parameters,
            status=TaskExecutionStep.StepStatus.RUNNING,
            started_at=datetime.now()
        )
        task.execution_history.append(step)
        
        try:
            # 模拟工具执行
            tool_result = self.simulate_tool_execution(tool_info)
            
            step.output_data = {
                "tool_name": tool_info.tool_name,
                "result": tool_result
            }
            step.status = TaskExecutionStep.StepStatus.COMPLETED
            step.completed_at = datetime.now()
            
            # 创建工具调用和结果消息
            tool_call = ToolCall(
                tool_call_id=f"call_{task.task_id}",
                tool_name=tool_info.tool_name,
                parameters=tool_info.tool_parameters
            )
            self.msg_manager.add_tool_call([tool_call])
            
            result_obj = ToolResult(
                result_id=f"result_{task.task_id}",
                tool_name=tool_info.tool_name,
                success=True,
                data=tool_result,
                output_type=ToolResult.OutputType.TEXT,
                metadata={},
                generated_at=datetime.now()
            )
            self.msg_manager.add_tool_result(result_obj, tool_call.tool_call_id)
            
        except Exception as e:
            # 异常必须有明确的异常信息
            error_info = TaskErrorInfo(
                error_type=type(e).__name__,
                error_message=str(e),
                error_details="工具执行过程中发生错误",
                stack_trace=None,
                occurred_at=datetime.now(),
                retryable=True
            )
            task.error_info = error_info
            task.status = Task.TaskStatus.FAILED
            
            step.error_info = error_info
            step.status = TaskExecutionStep.StepStatus.FAILED
            step.completed_at = datetime.now()
            
            self.msg_manager.add_error(str(e), type(e).__name__)
        
        return task
    
    def create_check_task(self, original_task: Task) -> Task:
        """
        创建检查任务（用于递归调用）
        """
        return Task(
            task_id=f"{original_task.task_id}_check",
            parent_task_id=original_task.task_id,
            task_group_id=None,
            input_data={
                "original_input": original_task.input_data,
                "tool_execution_result": "模拟工具结果"
            },
            output_data=None,
            status=Task.TaskStatus.PENDING,
            error_info=None,
            execution_history=[],
            created_at=datetime.now(),
            started_at=None,
            completed_at=None
        )
    
    def create_adjustment_task(self, original_task: Task,
                               check_result: Task) -> Task:
        """
        创建调整任务（用于递归调用）
        """
        return Task(
            task_id=f"{original_task.task_id}_adjust",
            parent_task_id=original_task.task_id,
            task_group_id=None,
            input_data={
                "original_task_status": original_task.status.value,
                "check_result": check_result.output_data
            },
            output_data=None,
            status=Task.TaskStatus.PENDING,
            error_info=None,
            execution_history=[],
            created_at=datetime.now(),
            started_at=None,
            completed_at=None
        )
    
    def execute_task_group_task(self, task: Task,
                                intent_result: IntentAnalysisResult) -> Task:
        """
        分支c: 任务组执行子流程
        """
        print(f"  → 任务组执行: {task.task_id}")
        
        group_info = intent_result.task_group_info
        
        # 创建任务组
        task_group = TaskGroup(
            task_group_id=f"group_{task.task_id}",
            parent_task_id=task.task_id,
            description=f"任务组: {task.task_id}",
            subtasks=[],
            status=TaskGroup.TaskGroupStatus.IN_PROGRESS,
            execution_mode=group_info.execution_mode,
            aggregated_result=None,
            created_at=datetime.now(),
            started_at=datetime.now(),
            completed_at=None
        )
        
        # 创建子任务
        for subtask_def in group_info.subtasks:
            subtask = Task(
                task_id=subtask_def.subtask_id,
                parent_task_id=task.task_id,
                task_group_id=task_group.task_group_id,
                input_data=subtask_def.input_data,
                output_data=None,
                status=Task.TaskStatus.PENDING,
                error_info=None,
                execution_history=[],
                created_at=datetime.now(),
                started_at=None,
                completed_at=None
            )
            task_group.subtasks.append(subtask)
        
        # 添加任务创建消息
        self.msg_manager.add_task_creation(task)
        
        # 执行子任务循环
        task_group.subtasks = self.execute_subtasks(task_group, group_info.execution_mode)
        
        # 聚合结果
        aggregated_result = self.aggregate_subtask_results(task_group.subtasks)
        task_group.aggregated_result = aggregated_result
        task_group.status = TaskGroup.TaskGroupStatus.COMPLETED
        task_group.completed_at = datetime.now()
        
        # 更新主任务
        task.output_data = {
            "task_group_id": task_group.task_group_id,
            "aggregated_result": aggregated_result,
            "subtask_count": len(task_group.subtasks)
        }
        
        self.msg_manager.add_task_completion(task, True, "任务组执行完成")
        
        return task
    
    def execute_subtasks(self, task_group: TaskGroup,
                        execution_mode: TaskGroupInfo.ExecutionMode) -> List[Task]:
        """
        递归执行子任务
        """
        completed_subtasks = []
        
        for subtask in task_group.subtasks:
            print(f"    → 执行子任务: {subtask.task_id}")
            
            self.msg_manager.add_task_update(subtask, f"开始执行子任务 {subtask.task_id}")
            
            # 递归调用任务流程
            completed_subtask = self.execute_task(subtask)
            completed_subtasks.append(completed_subtask)
            
            self.msg_manager.add_task_update(subtask, f"子任务 {subtask.task_id} 完成")
        
        return completed_subtasks
    
    def aggregate_subtask_results(self, subtasks: List[Task]) -> Dict[str, Any]:
        """
        聚合子任务结果
        """
        return {
            "subtask_results": [
                {
                    "task_id": st.task_id,
                    "status": st.status.value,
                    "output": st.output_data
                }
                for st in subtasks
            ],
            "summary": f"完成 {len(subtasks)} 个子任务"
        }
    
    def simulate_tool_execution(self, tool_info: SingleToolInfo) -> str:
        """
        模拟工具执行（实际应该调用真实工具）
        """
        return f"模拟执行 {tool_info.tool_name} 的结果"

        if result.output_data:

if __name__ == "__main__":
    example_recursive_task_execution()
```

#### 9.7.2 意图服务 (IntentService)

**职责**：负责分析任务输入，理解用户意图，选择合适的执行路径。

```python
@dataclass
class IntentService:
    """
    意图服务 - 分析任务意图并选择执行路径
    """
    
    llm_service: 'LLMInvocationService'
    
    def analyze_intent(self, task_input: Dict[str, Any]) -> IntentAnalysisResult:
        """
        分析任务意图
        - 理解任务目标
        - 分析输入信息
        - 判断执行路径（直接完成/单工具执行/任务组执行）
        """
        prompt = self.build_intent_analysis_prompt(task_input)
        response = self.llm_service.invoke(prompt)
        return self.parse_intent_response(response)
```

#### 9.7.3 基础执行服务 (BaseExecutionService)

**职责**：任务编排的核心能力，负责接收任务输入，进行意图分析，选择执行路径，并协调各子服务完成任务执行。

```python
@dataclass
class BaseExecutionService:
    """
    基础执行服务 - 任务编排的核心入口
    负责：意图分析 → 路径选择 → 服务协调 → 结果返回
    """
    
    intent_service: IntentService
    tool_task_executor: 'ToolTaskExecutor'
    task_group_executor: 'TaskGroupExecutor'
    
    def execute_task(self, task: Task) -> Task:
        """
        【任务编排入口】执行任务的完整流程
        """
        task.started_at = datetime.now()
        task.status = Task.TaskStatus.ANALYZING
        
        # 阶段1: 意图分析（调用意图服务）
        intent_result = self.intent_service.analyze_intent(task.input_data)
        
        # 阶段2: 根据意图选择执行路径并分发
        if intent_result.execution_path == IntentAnalysisResult.ExecutionPath.DIRECT_COMPLETION:
            # 分支a: 直接完成 - 无需调用工具或子任务
            return self.execute_direct_completion(task, intent_result)
        elif intent_result.execution_path == IntentAnalysisResult.ExecutionPath.SINGLE_TOOL:
            # 分支b: 单工具执行 - 调用工具任务执行服务
            return self.tool_task_executor.execute(task, intent_result)
        elif intent_result.execution_path == IntentAnalysisResult.ExecutionPath.TASK_GROUP:
            # 分支c: 任务组执行 - 调用任务组执行服务
            return self.task_group_executor.execute(task, intent_result)
```

#### 9.7.4 工具任务执行服务 (ToolTaskExecutor)

**职责**：负责执行需要调用工具的任务。包含三个关键子任务，其中**工具任务检测**和**工具任务调整**会**递归调用基础执行服务**。

```python
@dataclass
class ToolTaskExecutor:
    """
    工具任务执行服务 - 执行单工具任务流程
    包含：L2工具执行、工具任务检测、工具任务调整
    
    关键设计：
    - L2工具执行：调用L2层工具执行服务完成实际工具调用（不递归）
    - 工具任务检测：创建检测子任务，递归调用基础执行服务进行检查
    - 工具任务调整：创建调整子任务，递归调用基础执行服务进行调整决策
    """
    
    tool_execution_service: 'ToolExecutionService'  # L2工具执行服务
    base_executor: BaseExecutionService             # 用于递归调用
    
    def execute(self, task: Task, intent_result: IntentAnalysisResult) -> Task:
        """
        执行单工具任务的完整流程
        流程：L2工具执行 → 工具任务检测（递归） → 工具任务调整（递归）
        """
        max_iterations = 3
        iteration = 0
        
        while iteration < max_iterations:
            iteration += 1
            
            # ========== 子任务1: L2工具执行 ==========
            task = self.execute_l2_tool(task, intent_result.single_tool_info)
            
            if task.status == Task.TaskStatus.FAILED:
                break
            
            # ========== 子任务2: 工具任务检测（递归） ==========
            check_result = self.execute_task_check(task)
            
            # ========== 子任务3: 工具任务调整（递归） ==========
            adjust_result = self.execute_task_adjustment(task, check_result)
            
            if adjust_result.output_data.get("action") == "complete":
                task.output_data = {"result": "成功", "iterations": iteration}
                break
        
        return task
    
    def execute_task_check(self, task: Task) -> Task:
        """
        工具任务检测子任务 - 创建检测子任务并递归调用基础执行服务
        """
        check_task = Task(
            task_id=f"{task.task_id}_check",
            parent_task_id=task.task_id,
            input_data={
                "original_input": task.input_data,
                "tool_execution_result": self._get_last_tool_result(task)
            }
        )
        # 递归调用基础执行服务
        return self.base_executor.execute_task(check_task)
    
    def execute_task_adjustment(self, task: Task, check_result: Task) -> Task:
        """
        工具任务调整子任务 - 创建调整子任务并递归调用基础执行服务
        """
        adjust_task = Task(
            task_id=f"{task.task_id}_adjust",
            parent_task_id=task.task_id,
            input_data={
                "original_task_status": task.status.value,
                "check_result": check_result.output_data
            }
        )
        # 递归调用基础执行服务
        return self.base_executor.execute_task(adjust_task)
```

**工具任务执行服务的三个子任务关系**：

| 子任务 | 职责 | 是否递归调用基础执行服务 |
|--------|------|------------------------|
| **L2工具执行** | 调用L2层工具执行服务执行指定工具 | 否（直接调用） |
| **工具任务检测** | 创建检测子任务，检查工具执行结果 | **是** |
| **工具任务调整** | 创建调整子任务，决定下一步行动 | **是** |

#### 9.7.5 任务组执行服务 (TaskGroupExecutor)

**职责**：负责执行包含多个子任务的复杂任务组。**任务组执行服务本身按指定模式顺序执行子任务，而每个子任务内部会递归调用基础执行服务**。

```python
@dataclass
class TaskGroupExecutor:
    """
    任务组执行服务 - 执行任务组流程
    
    关键设计：
    - 任务组层面：按指定模式（顺序/并行/依赖）调度子任务执行顺序
    - 子任务层面：每个子任务通过 base_executor.execute_task() 递归调用完整任务编排流程
    - 结果聚合：收集所有子任务结果并聚合返回
    """
    
    base_executor: BaseExecutionService  # 用于子任务的递归调用
    
    def execute(self, task: Task, intent_result: IntentAnalysisResult) -> Task:
        """
        执行任务组任务
        流程：创建任务组 → 创建子任务 → 按模式顺序执行子任务 → 聚合结果
        """
        group_info = intent_result.task_group_info
        
        # 创建任务组和子任务定义
        task_group = TaskGroup(
            task_group_id=f"group_{task.task_id}",
            parent_task_id=task.task_id,
            subtasks=[...],
            execution_mode=group_info.execution_mode
        )
        
        # 【关键】按指定模式顺序执行子任务
        # 任务组执行服务负责调度顺序，子任务内部递归调用基础执行服务
        task_group.subtasks = self._execute_subtasks(task_group, group_info.execution_mode)
        
        # 聚合结果
        aggregated_result = self._aggregate_results(task_group.subtasks)
        
        return task
    
    def _execute_subtasks(self, task_group: TaskGroup,
                         execution_mode: TaskGroupInfo.ExecutionMode) -> List[Task]:
        """
        按指定模式顺序执行子任务
        
        【重要区别】
        - 任务组执行服务：调度子任务执行顺序
        - 子任务：通过 base_executor.execute_task() 递归调用完整任务编排流程
        """
        completed_subtasks = []
        
        if execution_mode == TaskGroupInfo.ExecutionMode.SEQUENTIAL:
            # 顺序执行
            for subtask in task_group.subtasks:
                completed_subtask = self.base_executor.execute_task(subtask)
                completed_subtasks.append(completed_subtask)
        elif execution_mode == TaskGroupInfo.ExecutionMode.PARALLEL:
            # 并行执行（简化实现）
            for subtask in task_group.subtasks:
                completed_subtask = self.base_executor.execute_task(subtask)
                completed_subtasks.append(completed_subtask)
        elif execution_mode == TaskGroupInfo.ExecutionMode.DEPENDENCY_BASED:
            # 基于依赖执行
            completed_subtasks = self._execute_dependency_based(task_group.subtasks)
        
        return completed_subtasks
```

```
                    用户请求
                         │
                         ↓
            ┌───────────────────────┐
            │   基础执行服务入口     │
            │  (execute_task)      │
            └───────────┬───────────┘
                        │
                        ↓
            ┌───────────────────────┐
            │     意图服务           │ ← 分析意图，选择路径

            └───────────┬───────────┘
                        │
         ┌──────────────┼──────────────┐
         │              │              │

   ┌─────────┐   ┌─────────────┐   ┌─────────────┐
   │直接完成 │   │工具任务执行 │   │任务组执行   │
   │         │   │ (ToolTask) │   │ (TaskGroup) │
   └─────────┘   └──────┬──────┘   └──────┬──────┘
                        │                 │
         ┌──────────────┼────────┐  ┌─────┴─────┐
         ↓              ↓        ↓  ↓           ↓
    ┌──────────┐ ┌──────────┐┌──────────┐┌──────────┐┌──────────┐
    │ L2工具   │ │工具任务  ││工具任务  ││ 创建任务 ││ 顺序执行 │
    │ 执行     │ │ 检测     ││ 调整     ││   组     ││ 子任务   │
    └──────────┘ └────┬─────┘└────┬─────┘└──────────┘└────┬─────┘
                     │            │                       │
                     └────┬───────┘                       │
                          │                               │
                          ↓                               ↓
                   ┌───────────────┐              ┌───────────────┐
                   │ 递归调用      │              │ 递归调用      │
                   │ base_executor │              │ base_executor │
                   │ .execute_task │              │ .execute_task │
                   └───────────────┘              └───────────────┘
```

**数据流说明**：
1. **输入**：用户请求 → Task → base_executor.execute_task()
2. **意图分析**：Task.input_data → IntentService.analyze_intent() → IntentAnalysisResult
3. **路径分发**：根据 execution_path 路由到对应执行服务
4. **工具执行**：ToolTaskExecutor.execute_l2_tool() → L2工具执行服务 → ToolResult
5. **递归检测**：创建检测子任务 → base_executor.execute_task(check_task)
6. **递归调整**：创建调整子任务 → base_executor.execute_task(adjust_task)
7. **任务组执行**：TaskGroupExecutor._execute_subtasks() → 顺序调度 → base_executor.execute_task(subtask)
8. **输出**：聚合结果 → Task.output_data → 返回给用户

**控制流说明**：
- **单一入口**：所有任务都通过 `BaseExecutionService.execute_task()` 进入
- **递归机制**：检测、调整子任务和任务组的子任务都会递归调用核心服务
- **分层职责**：
  - 基础执行服务：决策和路由
  - 工具任务执行服务：工具调用 + 递归检查调整
  - 任务组执行服务：子任务调度 + 递归执行

**任务编排服务框架**（第9.7节）详细定义了：
- **意图服务**: 分析任务意图，选择执行路径（直接完成/单工具执行/任务组执行）

- **任务组执行服务**: 按顺序/并行/依赖模式调度子任务，每个子任务内部递归调用基础执行服务

**核心设计原则**：
- **单一入口**：所有任务都通过 `BaseExecutionService.execute_task()` 进入编排流程
- **递归机制**：检测、调整子任务和任务组的子任务都会递归调用核心服务
- **分层职责**：基础执行服务负责决策和路由，工具任务执行服务负责工具调用和递归检查调整，任务组执行服务负责子任务调度和递归执行

这些概念通过清晰的边界和关系，构成了完整的领域模型，支持从简单对话到复杂任务组的各种场景，特别是通过递归机制实现了灵活且强大的任务处理能力。

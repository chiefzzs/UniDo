# L2 领域记忆与执行层 - 层总览

## 1. 层定位与职责

**L2 领域记忆与执行层**  提供l3 层基于业务的原子操作， 层内各服务相互关系比较少

### 1.1 核心职责
- 管理领域实体（Project、WorkspaceConfig、ModelConfig）
- 管理会话和对话记忆（Session、Dialog、Message、TaskGroup、Task）
- 执行LLM调用
  - 构造LLM 请求： 包含历史数据、包含工具定义数据
- 执行工具调用
- 管理工具和技能

### 1.2 设计原则
- **领域驱动**：基于DDD设计，明确聚合根和实体
- **事件驱动**：通过事件总线进行组件通信
- **职责分离**：每个服务只负责一个领域职责
- **依赖简化**： 服务之间依赖关系简单，避免循环依赖

## 7. 数据结构关系

### 7.1 聚合关系图

```
Session（会话）
    └── Dialog（对话）[*]
            ├── Message（消息）[*]
            └── TaskGroup（任务组）[1] ← 入口任务组
                    └── Task（任务）[*]
```

### 7.2 实体关系说明

| 实体 | 说明 | 关联关系 |
|-----|------|---------|
| **Session** | 会话，聚合多个对话 | 1:N → Dialog |
| **Dialog** | 对话，聚合多个消息 | N:1 ← Session, 1:N → Message, 1:1 → TaskGroup |
| **Message** | 消息 | N:1 ← Dialog |
| **TaskGroup** | 任务组，管理一组任务 | N:1 ← Dialog, 1:N → Task |
| **Task** | 任务 | N:1 ← TaskGroup |

### 7.3 消息格式定义（Message）

基于 LLM 请求格式，Message 实体包含以下字段：

| 字段 | 类型 | 说明 |
|-----|------|------|
| `id` | string | 消息唯一标识 |
| `role` | string | 角色：`system`、`user`、`assistant`、`tool` |
| `content` | string \| array | 消息内容，支持纯文本或多部分内容数组 |
| `content.type` | string | 内容类型：`text` |
| `content.text` | string | 文本内容 |
| `tool_calls` | array | 工具调用列表 |
| `tool_call_id` | string | 工具调用ID（用于工具返回消息） |
| `created_at` | timestamp | 创建时间 |

**消息内容结构示例：**
```json
{
  "role": "user",
  "content": [
    {
      "type": "text",
      "text": "用户输入的文本内容"
    }
  ]
}
```

### 7.4 工具定义格式（Tool）

工具定义采用 OpenAI 函数调用格式：

| 字段 | 类型 | 说明 |
|-----|------|------|
| `type` | string | 类型：`function` |
| `function.name` | string | 工具名称 |
| `function.description` | string | 工具描述（用于 LLM 理解） |
| `function.parameters` | object | 参数定义（JSON Schema 格式） |
| `function.parameters.type` | string | 固定为 `object` |
| `function.parameters.properties` | object | 参数属性定义 |
| `function.parameters.required` | array | 必填参数列表 |

**工具定义示例：**
```json
{
  "type": "function",
  "function": {
    "name": "SearchCodebase",
    "description": "搜索代码库中的相关代码",
    "parameters": {
      "type": "object",
      "properties": {
        "information_request": {
          "type": "string",
          "description": "需要查找的信息描述"
        },
        "target_directories": {
          "type": "array",
          "items": {
            "type": "string"
          },
          "description": "目标目录列表"
        }
      },
      "required": ["information_request"]
    }
  }
}
```

### 7.5 工具执行返回数据格式

工具执行完成后，返回结果以特定格式的消息形式存储：

| 字段 | 类型 | 说明 |
|-----|------|------|
| `role` | string | 固定为 `tool` |
| `content` | string | 工具执行结果（JSON 字符串或文本） |
| `tool_call_id` | string | 关联的工具调用ID |
| `tool_name` | string | 工具名称 |
| `status` | string | 执行状态：`success`、`error` |
| `error_message` | string | 错误信息（仅在状态为 error 时有值） |

**工具返回消息示例：**
```json
{
  "role": "tool",
  "content": "{\"results\": [...]}",
  "tool_call_id": "call_abc123",
  "tool_name": "SearchCodebase",
  "status": "success"
}
```

### 7.6 LLM 请求结构（LLMRequest）

| 字段 | 类型 | 说明 |
|-----|------|------|
| `model` | string | 模型名称，如 `Qwen/Qwen3.5-397B-A17B` |
| `max_completion_tokens` | integer | 最大生成 token 数 |
| `stream` | boolean | 是否流式响应 |
| `tools` | array | 工具定义列表（见 7.4） |
| `messages` | array | 消息列表（见 7.3） |

### 7.7 LLM 流式响应结构（LLMResponseChunk）

| 字段 | 类型 | 说明 |
|-----|------|------|
| `id` | string | 响应ID |
| `object` | string | 类型：`chat.completion.chunk` |
| `created` | integer | 创建时间戳 |
| `model` | string | 模型名称 |
| `choices` | array | 选择列表 |
| `choices[].index` | integer | 索引 |
| `choices[].delta.role` | string | 角色 |
| `choices[].delta.content` | string | 内容片段 |
| `choices[].delta.tool_calls` | array | 工具调用 |
| `choices[].delta.reasoning_content` | string | 思考过程内容 |
| `choices[].finish_reason` | string | 结束原因 |
| `usage` | object | 使用统计 |
| `usage.prompt_tokens` | integer | 提示词 token 数 |
| `usage.completion_tokens` | integer | 生成 token 数 |
| `usage.total_tokens` | integer | 总 token 数 |

## 8. 组件职责详细说明

### 8.1 L2a 项目与配置管理服务
- **Project管理**：项目的创建、查询、更新、删除、归档
- **WorkspaceConfig管理**：工作区配置的创建、查询、更新、删除
- **ModelConfig管理**：模型配置的创建、查询、更新、删除

- **配置验证**：验证配置ID的有效性

### 8.2 L2b 记忆与状态管理服务
- **Session管理**：会话的创建、查询、更新、删除、列出（按项目过滤）
- **Dialog管理**：对话的创建、查询、更新、删除、列出（按会话过滤）
- **Message管理**：消息的创建、查询、更新、删除、列出（按对话过滤）
- **TaskGroup管理**：任务组的创建、查询、更新、删除、设置入口任务组
- **Task管理**：任务的创建、查询、更新、删除、列出（按对话/任务组过滤）
- **状态机管理**：会话/对话状态流转
- **记忆管理**：短期记忆、长期记忆、记忆压缩

### 8.3 L2c 工具执行服务
- **工具调用执行**：执行工具调用
- **结果处理**：处理工具执行结果
- **错误重试**：工具执行失败重试

### 8.4 L2d LLM执行服务
- **LLM调用**：执行LLM请求（从L2e获取构造好的请求）
- **流式响应处理**：处理流式输出
- **工具调用解析**：解析LLM响应中的工具调用

### 8.5 L2e 请求构造服务
- **Prompt构建**：构建LLM请求的prompt
- **上下文格式化**：格式化上下文信息（从L2b获取）
- **工具描述集成**：从L2f获取工具描述
- **技能描述集成**：从L2g获取技能描述
- **参数配置**：配置LLM请求参数

### 8.6 L2f 工具管理服务
- **工具注册**：注册新工具（支持自动注册）
- **工具描述生成**：生成工具的JSON Schema描述
- **工具能力评估**：评估工具能力
- **技能集成**：将技能作为工具纳入统一管理（从L2g获取技能）
- **工具列表聚合**：聚合原生工具和技能，提供统一的工具列表

### 8.7 L2g 技能管理服务
- **技能定义**：定义技能（组合多个工具调用）
- **技能组合**：组合多个技能形成复杂能力
- **技能调用链编排**：编排技能执行顺序
- **技能工具化**：将技能封装为工具格式，供L2f集成

## 9. 与L1层的交互

### 9.1 与L1b 持久化服务
- L2a、L2b、L2f、L2g 使用L1b进行实体持久化
- 支持JSON文件存储和环境隔离

### 9.2 与L1c LLM接入服务
- L2d 使用L1c执行LLM调用
- 支持多种LLM提供商

### 9.3 与L1d 事件系统
- 所有L2组件通过L1d发布和订阅事件
- 支持事件驱动的组件通信

## 10. 与L3层的交互

### 10.1 提供给L3层的能力
- 项目和配置管理能力
- 会话和记忆管理能力（含Dialog层级）
- 工具和技能执行能力
- LLM调用能力

### 10.2 接收L3层的调用
- L3a 通用任务协调服务调用L2b、L2c、L2d、L2e、L2g
- L3b 四大对话场景调用L2b、L2d、L2e
- L3c UI操作场景调用L2a、L2b

## 11. 容错与恢复

### 11.1 数据完整性保障
- 使用事务确保操作原子性
- 验证实体引用的有效性
- 维护关系完整性（Session→Dialog→Message/TaskGroup）

### 11.2 错误处理
- 捕获并处理持久化异常
- 返回有意义的错误信息
- 发布错误事件

### 11.3 恢复策略
- 支持软删除，可恢复已删除实体
- 支持归档状态，可恢复已归档实体
- 记录操作历史，支持回滚

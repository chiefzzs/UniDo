### L3a 通用任务协调服务目录

L3a是**任务编排的核心层**，包含8个独立服务，每个服务只负责一件事：

| 服务名 | 职责 | 调用下层 |
|-------|------|---------|
| **DialogueService** | 对话管理，调用基础执行服务 | BaseExecutionService |
| **IntentService** | 意图分析，选择执行路径 | L2d LLM执行服务 |
| **BaseExecutionService** | 任务编排核心入口，协调执行 | IntentService + ToolTaskExecutor + TaskGroupExecutor |
| **ToolTaskExecutor** | 工具任务执行服务 | L2c工具执行服务 + CheckTaskService + AdjustTaskService |
| **CheckTaskService** | 工具任务检测服务 | BaseExecutionService（递归） |
| **AdjustTaskService** | 工具任务调整服务 | BaseExecutionService（递归） |
| **TaskExecutionService** | 任务执行服务 | BaseExecutionService（递归） |
| **TaskGroupExecutor** | 任务组执行服务 | TaskExecutionService |

### L3a 核心设计原则

- **单一入口**：所有对话都通过 `DialogueService.process_dialogue()` 进入
- **核心路由**：所有任务执行都通过 `BaseExecutionService.execute_task()` 路由
- **递归机制**：检测、调整服务和任务执行服务都会递归调用基础执行服务
- **分层职责**：
  - DialogueService：对话管理
  - BaseExecutionService：决策和路由
  - ToolTaskExecutor：工具调用 + 检测调整
  - TaskGroupExecutor：子任务调度
  - TaskExecutionService：任务执行代理

### L3a 数据流说明

1. **输入**：用户请求 → DialogueService.process_dialogue()
2. **对话处理**：创建Task → BaseExecutionService.execute_task()
3. **意图分析**：Task.input_data → IntentService.analyze_intent() → IntentAnalysisResult
4. **路径分发**：根据execution_path路由到对应执行服务
5. **工具执行**：ToolTaskExecutor.execute() → L2c工具执行服务 → CheckTaskService → AdjustTaskService
6. **递归检测**：CheckTaskService.execute() → BaseExecutionService.execute_task(check_task)
7. **递归调整**：AdjustTaskService.execute() → BaseExecutionService.execute_task(adjust_task)
8. **任务组执行**：TaskGroupExecutor.execute() → TaskExecutionService.execute() → BaseExecutionService.execute_task(subtask)
9. **输出**：聚合结果 → DialogueResponse → 返回给用户

## 2. 历史记忆添加流程

历史消息序列是LLM对话的上下文基础，通过事件驱动的方式逐步构建完整的消息序列。详细设计请参考：[历史记忆添加流程设计.md](历史记忆添加流程设计.md)

### 2.1 消息类型定义

| 消息类型 | 角色(role) | 说明 |
|---------|-----------|------|
| 系统提示 | system | 会话初始化时添加，包含系统指令和工具定义 |
| 用户输入 | user | 用户文本输入 |
| 助手响应 | assistant | LLM生成的文本响应 |
| 思考过程 | assistant(think) | LLM的内部思考 |
| 工具调用 | assistant(tool_call) | 工具选择和参数 |
| 工具结果 | tool_result | 工具执行输出 |
| 任务上下文 | system(task) | 任务元信息 |

### 2.2 消息序列追加规则

| 事件 | 触发组件 | 追加位置 |
|-----|---------|---------|
| session.created | L2a | 序列开头 |
| message.created(user) | L2a | 序列末尾 |
| text_chunk.received | L2d | 序列末尾 |
| think_chunk.received | L2d | 序列末尾 |
| tool_selected | L2d | 序列末尾 |
| tool_output.received | L2c | 工具调用后 |
| task.created | L3a | 任务执行前 |

### 2.3 历史记忆流程与架构的对应关系

| 流程阶段 | L3层组件 | L2层组件 | L1层组件 |
|---------|---------|---------|---------|
| 会话初始化 | L3c | L2a、L2b | L1d |
| 用户输入 | L3a.DialogueService | L2a、L2b | L1d |
| 意图分析 | L3a.IntentService | L2d、L2b | - |
| LLM响应 | L3a.BaseExecutionService | L2d、L2a、L2b | L1c、L1d |
| 工具调用 | L3a.ToolTaskExecutor | L2c、L2a、L2b | - |
| 任务执行 | L3a.TaskGroupExecutor | L2a、L2b | - |
| 对话结束 | L3a.DialogueService | L2a、L2b | L1b、L1d |

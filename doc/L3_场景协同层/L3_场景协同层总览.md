# L3 场景协同层 - 层总览

## 1. 层定位与职责

**L3 场景协同层** 是系统的场景编排层，负责实现具体的业务场景和用户交互流程。

### 1.1 核心职责
- 实现任务编排核心能力（L3a）
- 处理UI操作场景（L3c）
- 协调跨场景的数据流和控制流

### 1.2 设计原则
- **场景驱动**：每个场景有独立的处理逻辑
- **统一接口**：所有场景使用相同的调用模式
- **可扩展性**：支持新增场景类型
- **事件驱动**：通过事件总线进行场景间通信

## 3. L3a 通用任务协调服务目录

L3a是**任务编排的核心层**，包含8个独立服务：

| 服务名 | 职责 | 调用下层 |
|-------|------|---------|
| **DialogueService** | 对话管理，调用基础执行服务 | BaseExecutionService |
| **IntentService** | 意图分析，选择执行路径 | DialogueBasedLLMService |
| **DialogueBasedLLMService** | 基于对话的LLM调用服务 | L2d LLM执行服务 + L2b记忆服务 + L2f工具管理服务 |
| **BaseExecutionService** | 任务编排核心入口，协调执行 | IntentService + ToolTaskExecutor + TaskGroupExecutor |
| **ToolTaskExecutor** | 工具任务执行服务 | L2c工具执行服务 + CheckTaskService + AdjustTaskService |
| **CheckTaskService** | 工具任务检测服务 | BaseExecutionService（递归） |
| **AdjustTaskService** | 工具任务调整服务 | BaseExecutionService（递归） |
| **TaskExecutionService** | 任务执行服务 | BaseExecutionService（递归） |
| **TaskGroupExecutor** | 任务组执行服务 | TaskExecutionService |

### 3.2 核心设计原则

- **单一入口**：所有对话都通过 `DialogueService.process_dialogue()` 进入
- **核心路由**：所有任务执行都通过 `BaseExecutionService.execute_task()` 路由
- **递归机制**：检测、调整服务和任务执行服务都会递归调用基础执行服务
- **分层职责**：
  - DialogueService：对话管理
  - BaseExecutionService：决策和路由
  - ToolTaskExecutor：工具调用 + 检测调整
  - TaskGroupExecutor：子任务调度
  - TaskExecutionService：任务执行代理

## 4. L3c UI操作场景服务详细说明

### 4.1 SessionManager（会话管理组件）

**职责**：管理用户会话的创建、查询、更新、删除和归档。

**支持场景**：
1. **会话创建**
   - 新建session对象
   - 初始化此session对象包含的dialog对象
   - 给第一个dialog对象增加system message，用于引导对话方向
   - 结果：
     - `sessions.json`中多了一个session对象，包含会话ID、项目ID、dialog ID等信息
     - `events.json`中多了一个event对象，包含会话创建事件
     - `dialogs.json`中多了一个dialog对象，包含对话ID、会话ID、对话名称、对话状态、对话消息等信息
     - `messages.json`中多了一个message对象，包含消息ID、对话ID、用户输入、系统回复等信息

2. **新建对话对象**
   - 如果是第一次，返回session对象的第一个dialog对象
   - 如果不是第一次，session对象新建一个dialog对象，并放到自己的队列中，返回新dialog对象

3. **得到历史消息**
   - **非压缩式获得**：
     - 从session对象中获取所有dialog对象
     - 从每个dialog对象中获取所有message对象
     - 合并所有message对象，返回给用户
   - **压缩式获得**：
     - 当历史消息 > 设定的上限
     - 最近的dialog对象，读取所有消息
     - 历史dialog，读取压缩消息

4. **得到大模型信息**
   - 依据本Session关联的Project，得到关联的大模型信息，返回大模型信息

**状态**：
- `idle`：首次创建的对象
- `ongoing`：对话进行中
- `finished`：对话结束

### 4.2 DialogManager（对话管理组件）

**职责**：管理对话的生命周期和状态流转。

**主要流程**：
1. 向session对象得到一个新的dialog对象
2. 生成一个用户输入的message对象，保存到此dialog对象的消息列表中
3. 调用通用任务协调流程
4. 结束会话，启动对话压缩历史
   - 压缩本dialog的所有消息，生成压缩消息对象
   - 保存压缩消息对象

### 4.3 MessageManager（消息管理组件）

**职责**：支持创建历史消息对象。

**支持创建的消息类型**：
1. **System Message**：用于引导对话方向
2. **User Message**：用于用户输入
3. **Assistant Message**：用于系统回复
4. **Tool Message**：用于工具调用结果

**消息类型报文格式**：

**System Message**：

```json
  "created_at": "2026-05-19T10:00:00Z"

```

**User Message**：

### 5.3 四大核心场景说明

#### SC04: 简单单次文本对话
- **特征**：无工具调用，直接LLM返回文本
- **执行路径**：DIRECT_COMPLETION
- **数据流**：用户输入 → IntentService → DialogueBasedLLMService → L2d LLM执行服务 → 助手回复
- **关键组件**：SessionManager, DialogManager, MessageManager, IntentService, DialogueBasedLLMService

#### SC05: 简单单次工具对话
- **特征**：LLM选择并调用单个工具，返回结果
- **执行路径**：SINGLE_TOOL
- **数据流**：用户输入 → IntentService → ToolTaskExecutor → L2c工具执行服务 → 工具结果 → 助手回复
- **关键组件**：SessionManager, DialogManager, MessageManager, IntentService, ToolTaskExecutor

#### SC16: 任务组对话
- **特征**：拆解为多个任务，顺序/并行执行
- **执行路径**：TASK_GROUP
- **数据流**：用户输入 → IntentService → TaskGroupExecutor → TaskExecutionService → 子任务执行 → 结果聚合
- **关键组件**：SessionManager, DialogManager, MessageManager, IntentService, TaskGroupExecutor, TaskExecutionService

#### SC17: 嵌套任务组对话
- **特征**：任务组中嵌套子任务组，递归执行
- **执行路径**：TASK_GROUP（嵌套）
- **数据流**：用户输入 → IntentService → TaskGroupExecutor → 递归TaskGroupExecutor → 自底向上聚合
- **关键组件**：SessionManager, DialogManager, MessageManager, IntentService, TaskGroupExecutor, TaskExecutionService

## 7. 关键设计决策

### 7.1 双层入口模式

**对话入口**：所有对话通过 `DialogueService.process_dialogue()` 进入

```python
class DialogueService:
    def __init__(self):
        self.base_executor = BaseExecutionService()
    
    def process_dialogue(self, session_id: str, user_input: str) -> DialogueResponse:
        task = Task(...)
        result = self.base_executor.execute_task(task)
        return DialogueResponse(result)
```

**任务执行入口**：所有任务执行通过 `BaseExecutionService.execute_task()` 路由

```python
class BaseExecutionService:
    def execute_task(self, task: Task) -> Task:
        intent_result = self.intent_service.analyze_intent(task.input_data)
        if intent_result.execution_path == DIRECT_COMPLETION:
            return self.execute_direct_completion(task, intent_result)
        elif intent_result.execution_path == SINGLE_TOOL:
            return self.tool_task_executor.execute(task, intent_result)
        elif intent_result.execution_path == TASK_GROUP:
            return self.task_group_executor.execute(task, intent_result)
```

### 7.2 递归调用机制

- CheckTaskService → BaseExecutionService.execute_task()
- AdjustTaskService → BaseExecutionService.execute_task()
- TaskExecutionService → BaseExecutionService.execute_task()

### 7.3 UI状态同步

- 使用事件驱动保持UI状态同步
- 定期同步状态到后端
- 支持离线状态恢复

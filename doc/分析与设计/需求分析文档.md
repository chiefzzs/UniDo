# 智能对话助手系统 - 需求分析文档

---

## 1. 需求背景

### 1.1 业务背景

本系统是一个基于大语言模型（LLM）的智能对话助手应用，旨在为用户提供自然语言交互能力，支持任务执行、代码开发、文档创作等多种场景。系统采用分层架构设计，具备良好的可扩展性和可维护性。

### 1.2 项目目标

- 提供流畅的对话体验
- 支持多场景任务执行（代码开发、创意写作、数据处理等）
- 实现完整的记忆和状态管理
- 提供可扩展的工具调用系统
- 支持多平台部署和多模型切换

## 3. 用户痛点问题

### 3.1 当前系统的问题

| 痛点编号 | 问题描述 | 影响范围 | 严重程度 |
|---------|---------|---------|---------|
| P001 | 工具调用失败时仅返回模拟成功消息，无法真正执行操作 | 所有工具调用场景 | 高 |
| P002 | ToolResult对象属性访问错误（使用`content`而非`result`） | WebSocket消息返回 | 高 |
| P003 | 工具查找失败后无法回退到配置驱动注册表 | 工具执行 | 中 |
| P004 | 工具执行结果格式不一致，难以统一处理 | 结果解析 | 中 |
| P005 | 缺少完整的工具执行验证测试 | 质量保障 | 中 |

### 3.2 用户体验痛点

- **工具调用不透明**：用户无法确认工具是否真正执行
- **错误反馈不及时**：工具执行失败时缺少明确的错误信息
- **执行结果不可追溯**：工具调用记录不完整，难以排查问题
- **跨平台兼容性差**：不同操作系统下工具行为不一致

## 4. 新工作流设计

### 4.1 工具执行工作流

```
用户请求 → 意图识别 → 工具选择 → 工具查找 → 参数验证 → 真实执行 → 结果返回 → 持久化
              ↓              ↓              ↓
         L3a意图服务    L2f工具管理    L2c工具执行
              ↓              ↓              ↓
         解析用户意图    获取工具定义    调用真实实现
```

### 4.2 工作流步骤说明

| 步骤 | 组件 | 职责 | 输入 | 输出 |
|-----|------|------|------|------|
| 1 | 用户界面 | 接收用户请求 | 用户输入文本 | 请求对象 |
| 2 | L3a意图服务 | 分析用户意图 | 用户输入 | 意图分析结果 |
| 3 | L3a任务协调 | 确定执行路径 | 意图结果 | 工具调用指令 |
| 4 | L2f工具管理 | 获取工具定义 | 工具名称/ID | 工具配置 |
| 5 | L2c工具执行 | 执行工具调用 | 工具参数 | 执行结果 |
| 6 | L1b持久化 | 存储调用记录 | 调用信息 | 存储确认 |
| 7 | L1d事件总线 | 发布执行事件 | 事件数据 | 事件确认 |
| 8 | 用户界面 | 展示执行结果 | 结果数据 | 可视化输出 |

## 5. 新旧工作流对比

### 5.1 工具执行流程对比

| 对比维度 | 旧工作流 | 新工作流 |
|---------|---------|---------|
| **工具查找** | 仅使用工具注册表 | 支持配置驱动注册表回退 |
| **执行方式** | 模拟执行（找不到工具时返回假成功） | 真实执行（找不到工具时返回错误） |
| **结果属性** | 使用`content`属性 | 使用`result`属性 |
| **错误处理** | 静默失败 | 明确错误信息 |
| **跨平台支持** | 有限 | 完整支持Windows/Linux/macOS |

### 5.2 数据流向对比

**旧工作流：**
```
用户请求 → L3 → L2c → （工具查找失败）→ 返回模拟成功
```

**新工作流：**
```
用户请求 → L3 → L2c → （工具查找失败）→ 尝试配置注册表 → 真实执行 → 返回真实结果
```

## 7. 详细场景描述

### 7.1 场景SC017：代码开发 - 工具执行

**触发界面**：对话界面输入框

**输入**：
| 参数 | 类型 | 描述 |
|-----|------|------|
| user_input | str | 用户输入的代码开发请求 |
| session_id | str | 当前会话ID |
| project_id | str | 当前项目ID |

**输出**：
| 参数 | 类型 | 描述 |
|-----|------|------|
| tool_result | ToolResult | 工具执行结果 |
| task_status | TaskStatus | 任务状态 |
| execution_history | list | 执行历史记录 |

**数据流向**：

```
1. 用户输入 → L5对话界面 → L4b WebSocket网关
2. WebSocket请求 → L3b场景服务 → 意图识别
3. 意图分析 → 确定工具调用 → L3a任务协调
4. 工具调用指令 → L2c工具执行服务
5. L2c查找工具 → L2f工具管理服务
6. 工具查找失败 → 尝试配置驱动注册表
7. 找到工具 → 执行真实工具实现
8. 执行结果 → 返回L3a → 更新任务状态
9. 任务状态 → 持久化到L1b
10. 执行事件 → 发布到L1d事件总线
11. 结果 → 返回L4b → 推送至L5界面
```

**持久化数据**：

```json
// tool_calls.json
{
  "call_id": "call-abc123",
  "tool_id": "T12",
  "tool_name": "RunCommand",
  "dialog_id": "dialog-123",
  "task_id": "task-456",
  "input_params": {"command": "mkdir test_dir"},
  "output_result": "命令执行成功",
  "status": "completed",
  "start_time": "2026-05-20T10:30:00",
  "end_time": "2026-05-20T10:30:01",
  "duration": 1.0
}

// tasks.json
{
  "task_id": "task-456",
  "input_data": {"user_input": "创建测试目录"},
  "status": "completed",
  "output_data": {"result": "成功", "iterations": 1},
  "execution_history": [
    {"step": "tool_execution", "tool_name": "RunCommand", "result": "成功"}
  ]
}
```

**异常情况**：

| 异常类型 | 触发条件 | 处理方式 |
|---------|---------|---------|
| 工具不存在 | 工具名称/ID未注册 | 返回错误：`Tool not found: {tool_name}` |
| 参数错误 | 缺少必需参数 | 返回错误：`Missing required parameter: {param}` |
| 执行失败 | 工具执行过程出错 | 返回错误：执行异常信息 |
| 超时 | 工具执行超过时间限制 | 返回错误：`Tool execution timeout` |

## 9. 领域模型

### 9.1 核心实体关系

```
Project (聚合根)
  │
  ├─ sessions: List[Session]
  │
  ├─ workspace_id ──→ WorkspaceConfig
  │
  └─ model_config_id ──→ ModelConfig

Session (聚合根)
  │
  ├─ messages: List[Message]
  │
  └─ task_group_id ──→ TaskGroup (可选)

TaskGroup
  │
  ├─ tasks: List[Task]
  │
  └─ execution_mode: ExecutionMode

Task
  │
  ├─ tool_name: str
  │
  ├─ parameters: Dict[str, Any]
  │
  └─ execution_history: List[ExecutionStep]

ToolCall
  │
  ├─ tool_id: str
  │
  ├─ tool_name: str
  │
  ├─ input_params: Dict[str, Any]
  │
  ├─ output_result: str
  │
  └─ status: CallStatus

Message
  │
  ├─ role: user/assistant/tool
  │
  ├─ content: str
  │
  └─ tool_call_id: Optional[str]
```

### 9.2 架构层级划分

| 层级 | 名称 | 职责 | 服务组件 |
|-----|------|------|---------|
| L1 | 基础设施层 | 基础服务支撑 | 持久化、LLM、事件、配置 |
| L2 | 领域记忆与执行层 | 业务逻辑处理 | 实体管理、记忆状态、工具执行、LLM执行 |
| L3 | 场景协同层 | 场景编排协调 | 任务协调、场景服务、UI操作场景 |
| L4 | 网关层 | 对外接口管理 | HTTP API、WebSocket、认证授权 |
| L5 | UI应用层 | 用户界面展示 | 对话界面、侧边栏、配置面板、任务面板 |

### 9.3 服务分类

**原子服务（高内聚低耦合）**：
- L1b_persistence：持久化服务
- L1c_llm：LLM基础设施
- L1d_events：事件总线
- L2a_project_config：项目配置
- L2b_memory_state：记忆状态管理

**控制服务（服务聚合）**：
- L2c_tool_execution：工具执行调度
- L2d_llm_execution：LLM执行协调
- L3a_task_coordination：任务协调
- L3b_scene_service：场景服务
- L4_gateway：网关聚合

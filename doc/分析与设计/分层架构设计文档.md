# 分层架构设计文档

## 1. 架构概述

本系统采用四层架构设计，从上到下依次为：
- **L4 网关层 (Gateway Layer)**：对外提供 HTTP/WebSocket 接口
- **L3 场景协调层 (Scenario Coordination Layer)**：业务流程编排和协调
- **L2 领域层 (Domain Layer)**：核心业务逻辑和实体管理
- **L1 基础设施层 (Infrastructure Layer)**：通用能力和底层服务

## 2. 各层级服务详细说明

### 2.1 L1 基础设施层 (Infrastructure Layer)

**职责**：提供系统级的基础能力，被上层服务依赖，不直接参与业务逻辑。

|------|----------|----------|
| **L1b Persistence** | `L1b_persistence/` | 数据持久化存储，文件操作 |
| - StorageFactory | `storage_factory.py` | 存储工厂，创建存储实例 |
| - FileStorage | `file_storage.py` | 文件系统存储实现 |
| - PersistenceService | `persistence_service.py` | 通用持久化接口 |
| **L1c LLM Client** | `L1c_llm/` | LLM API 调用适配 |
| - LLMClient | `llm_client.py` | LLM客户端主类 |
| - BaseAdapter | `api_adapters/base_adapter.py` | 适配器基类 |
| - OpenAIAdapter | `api_adapters/openai_adapter.py` | OpenAI API适配器 |
| - AnthropicAdapter | `api_adapters/anthropic_adapter.py` | Anthropic API适配器 |
| - QwenAdapter | `api_adapters/qwen_adapter.py` | 通义千问API适配器 |
| - RequestConstructor | `request_constructor.py` | 请求构造器 |
| - ResponseParser | `response_parser.py` | 响应解析器 |
| **L1d Event Bus** | `L1d_events/` | 事件驱动通信 |
| - EventBus | `event_bus.py` | 事件总线核心 |

```

                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────────────────┐

│ ┌─────────────────────────────────────────────────────────────────────────────────┐   │
│ │ WebSocketServer._handle_send_message()                                           │   │
│ │   1. 解析用户消息                                                                │   │
│ │   2. 创建 Dialog (dialog_id)                                                   │   │
│ │   3. 发布 message.created 事件                                                  │   │

                                      │

│ ┌─────────────────────────────────────────────────────────────────────────────────┐   │
│ │ DialogueService.process_message()                                                │   │
│ │   1. 保存用户消息到 L2b (Message role=user)                                     │   │

│ │   输出: IntentResult(intent_type, execution_path)                                │   │
│ │   - SINGLE_TOOL: 需要工具调用                                                   │   │
│ │   - TASK_GROUP: 需要任务组执行                                                   │   │

│ │   1. 从 L2b 获取对话历史 messages[]                                              │   │
│ │   2. 调用 L2e RequestBuilder 构建请求                                            │   │
│ │   3. 调用 LLMExecutionService.execute()                                         │   │

│ │   1. 调用 L1c LLMClient 发送请求                                               │   │
│ │   2. 解析 LLM 响应 (content, tool_calls)                                       │   │
│ │   3. 保存 LLMCallRecord                                                         │   │
│ │   4. 返回 LLMExecutionResponse                                                  │   │
│ └─────────────────────────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────┬───────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────────────┐

│ │   输出: tool_results=[{role: tool, tool_call_id, content}, ...]                 │   │

### 3.2 控制流在不同层级的流转

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                                 控制流顺序                                                │
├─────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                         │
│  1. L4 网关层                                                                        │
│     ├── 接收 WebSocket 消息                                                            │
│     ├── 验证消息格式                                                                   │
│     └── 转发到 L3 层                                                                  │
│                                                                                         │
│  2. L3 场景协调层                                                                    │
│     ├── DialogueService: 协调整个对话流程                                              │
│     │   ├── 创建 Dialog                                                               │
│     │   ├── 保存用户消息                                                               │
│     │   └── 调用 BaseExecutionService                                                 │
│     │                                                                                 │
│     ├── IntentService: 分析用户意图                                                   │
│     │   ├── 判断执行路径 (SINGLE_TOOL / TASK_GROUP / DIRECT_COMPLETION)               │
│     │   └── 返回 IntentResult                                                         │
│     │                                                                                 │
│     ├── BaseExecutionService: 执行任务                                                 │
│     │   ├── 根据执行路径分发                                                           │
│     │   │   ├── DIRECT_COMPLETION → 直接返回                                          │
│     │   │   ├── SINGLE_TOOL → ToolTaskExecutor                                       │
│     │   │   └── TASK_GROUP → TaskGroupExecutor → TaskExecutionService                │
│     │   │                                                                             │
│     │   └── 管理 messages[] 数组                                                      │
│     │                                                                                 │
│     └── ToolGroupExecutor: 执行工具组                                                 │
│         ├── 遍历工具调用列表                                                           │
│         ├── 调用 ToolTaskExecutor                                                     │
│         └── 收集工具结果                                                               │
│                                                                                         │
│  3. L2 领域层                                                                        │
│     ├── DialogueBasedLLMService: LLM 调用编排                                          │
│     │   ├── 获取对话历史                                                               │
│     │   ├── 调用 RequestBuilder                                                       │
│     │   └── 调用 LLMExecutionService                                                  │
│     │                                                                                 │
│     ├── LLMExecutionService: 执行 LLM 请求                                            │
│     │   ├── 调用 L1c LLMClient                                                       │
│     │   ├── 解析响应                                                                   │
│     │   └── 返回结果                                                                   │
│     │                                                                                 │
│     └── ToolExecutor: 执行单个工具                                                    │
│         ├── 参数验证                                                                   │
│         ├── 获取工具实现                                                               │
│         └── 执行并返回结果                                                             │
│                                                                                         │

## 8. 总结

### 8.1 架构特点

1. **分层清晰**：每层有明确的职责边界，上层依赖下层
2. **事件驱动**：通过 EventBus 实现服务间解耦通信
3. **配置驱动**：通过 ConfigDrivenToolRegistry 实现工具自动发现
4. **领域模型**：核心实体(Session, Dialog, Message, Task)定义清晰

### 8.2 关键设计模式

1. **工厂模式**：StorageFactory 创建存储实例
2. **适配器模式**：LLM 客户端使用适配器支持多 API
3. **注册表模式**：工具注册和发现
4. **策略模式**：不同执行路径(SINGLE_TOOL, TASK_GROUP, DIRECT_COMPLETION)

### 8.3 数据流特点

1. **消息驱动**：对话通过 Message 实体在服务间传递
2. **状态累积**：messages[] 数组累积对话历史
3. **工具结果转换**：ToolResult 转换为 role=tool 消息
4. **递归执行**：工具执行后递归调用 LLM 直到完成
